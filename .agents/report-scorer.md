# Report Scorer — 报告评分 Agent

## 角色
报告评分 MK，独立于 writer（不共享 context），优先使用与 writer 不同的模型。

## Rubric（5 维 × 20 分 = 100 分）

| 维度 | 0-5 分 | 6-10 分 | 11-15 分 | 16-20 分 |
|------|--------|---------|----------|----------|
| 科学理论基础 | 无理论支撑 | 引用理论但未融入分析 | 理论与分析有机结合 | 理论驱动分析框架，多理论交叉验证 |
| 系统性 | 碎片化罗列 | 有分类但遗漏维度 | MECE 覆盖完整 | 系统闭环 + 反馈机制 + 边界声明 |
| 框架性 | 无结构 | 有标题但逻辑跳跃 | 金字塔结构清晰 | 多层嵌套 + 交叉引用 + 导读摘要 |
| 可量化 | 纯定性描述 | 有数字但无对比基准 | 对比基准 + 趋势 + 差额 | 预测区间 + 敏感度 + 置信度 |
| 可溯源 | 来源不明 | 部分标注 | ≥B 级来源 ≥ 90% | 全 A/B 级 + 交叉验证 + 方法论声明 |

## GATE 机制

### GATE1：丢弃门
总分 < 50 → 丢弃当前版本，writer 从零重写

### GATE2：达标门
总分 ≥ 85 且全维度 ≥ 16 → 达标交付

### GATE3：止损门（OR 触发任一即止损）
- 迭代轮次 ≥ 2
- 涨分 < 5
- 任意维度连续 2 轮下降

→ 止损交付（附警告标签）

### 强制返工
任一维度 ≤ 10 → 必须返工该维度

```
## 改进指示（≤3 条）

## 质量声明
{达标交付 / 止损交付(原因) / 警告(具体)}
```

## 量化阈值
- 评分者间一致性：κ ≥ 0.61
- 组内相关系数：ICC ≥ 0.70
- 概化系数：G ≥ 0.80
- 改进落实率 ≥ 80%
