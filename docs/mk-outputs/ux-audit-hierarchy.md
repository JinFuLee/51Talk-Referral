### 页面 1：运营总览（首页）
**路径**：`frontend/app/page.tsx`

#### 信息层级评估
- **KPI 突出度**：⭐⭐⭐
  - 岗位筛选器、时间进度条、KPI 卡片网格、KPI 8 项、漏斗卡、月度达成、D2b 基准、数据源——8 个区块同级堆叠，用户扫视成本高
  - StatCard 主数值 `text-xl`，尚可辨识，但缺少页级核心 KPI（如"本月业绩"或"付费达成率"）单独放大突出
  - 时间进度条放在筛选器下方，位置合理；但进度百分比仅为 `text-sm font-semibold`，视觉权重不足以警示落后
- **数据扁平度**：8 个区块权重接近，用户无法 3 秒内确认"今天最重要的数字是什么"
- **3 秒焦点**：不能——KPI 卡片网格（2~3 列）与 KPI 8 项卡片（更密集）并列，主次不分

#### 指标完整度

| 指标类型 | CLAUDE.md 要求 | 当前实现 | 缺失项 |
|---------|-------------|---------|--------|
| 数值类（注册/付费/业绩） | 8 项全量 | KPI 8 项已实现（`KPI8Card` 组件，8 行 grid） | 无缺失，已合规 |
| 效率类（注册→预约转化率等） | 5 项全量 | `FunnelSnapshot` 仅展示转化率 + 进度条；`RateCard8` 展示实际率/目标率/目标差 | **损失量化（第 4 项）、根因标注（第 5 项）缺失** |
| D2b 基准（参与率/打卡率） | 5 项 | 仅展示实际率，无目标率、无目标差、无损失量化、无根因 | **4 项缺失** |

#### 布局改善建议

| 区域 | Before（当前状态） | After（改善目标） | 优先级 |
|------|----------------|----------------|--------|
| 页面顶部英雄区 | 无醒目 Hero KPI，标题 `text-lg` 与卡片数字同级 | 增加 Hero 区：本月业绩（大字 `text-4xl`）+ 达成率进度圈（放大至 96px）+ 落后/超额状态标签 | P0 |
| KPI 8 项区块 | 紧接 KPI 卡片网格之后，整体视觉对比度弱 | KPI 8 项折叠为「展开详情」交互，默认只显示绝对差和进度差两列 | P1 |
| 效率类 FunnelSnapshot | 仅有进度条，无损失量化 | 在每条转化率下方增加红色损失估算行（`gap < 0 → 损失 N 个付费 → $M`） | P0 |
| D2b 基准 | 7 个纯数字格子，无目标对比 | 每格增加目标值和达成状态色点（绿/黄/红） | P1 |
| 数据源状态 | 置于页面最底部，重要性等同于 KPI | 移至折叠面板或页面角落 badge，不占主流位置 | P2 |

## 全站信息架构问题模式

### 模式 1：数据扁平——等宽等权

**表现**：表格所有列 `text-xs` + 等宽，读者没有视觉锚点。
**受影响页面**：围场矩阵、漏斗表格、渠道分析（重度）。
**通用解法**：
```
主列（核心 KPI）: text-sm font-bold + 加宽
次列（支撑数据）: text-xs text-muted + 正常宽度
隐藏列（详情）: 折叠 / tooltip
```

### 模式 2：效率指标仅展示实际值

**表现**：打卡率/参与率/触达率只有一个数字，无目标基准和损失链。
**受影响页面**：全站 6 个以上页面的效率型指标。
**通用解法**：抽取 `EfficiencyCard5` 组件复用（实际率 + 目标率 + 目标差 + 损失链 + 根因）。

### 模式 3：3 秒焦点缺失

**表现**：页面无明确的 Hero 区或第一视觉锚点，用户需要扫描整页。
**受影响页面**：首页（8 区块堆叠）、漏斗页（4 区块等级）、渠道页（4 Tab 等级）。
**通用解法**：每页顶部设置 1~3 个「大字英雄数字」，字号 `text-3xl` + 颜色状态 + 配套目标进度。

## 推荐优先实施顺序

```
第一阶段（P0 修复，高影响）
  → 首页漏斗 FunnelSnapshot：添加损失量化行
  → 打卡管理：添加目标率 + 差值
  → 漏斗分析：柱状图添加目标基准线

第二阶段（全站 EfficiencyCard5 组件化）
  → 抽取通用效率卡片组件（5 项合规）
  → 替换围场/打卡/触达监控的效率展示

第三阶段（布局英雄化）
  → 首页添加 Hero 区（业绩/付费达成的大字）
  → 围场矩阵列优先级改造
```
