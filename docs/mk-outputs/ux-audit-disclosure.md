# UX 渐进式披露审计报告

**审计范围**：ref-ops-engine 全站 32 个页面 + 核心组件
**设计规范基准**：`09-progressive-disclosure.md` v2.0（L0+L1 默认，L2 复杂场景按需）
**审计日期**：2026-03-25

---

## 核心发现（执行摘要）

> **全站 L1 覆盖率约 15%。** BrandDot 组件已定义但被 0 个页面使用。
> StatMiniCard / RateCard / StatCard 三个最高频组件均无 L1 副标题解释。
> 32 个页面中，约 28 个页面存在"数字有颜色但无解释"的 L0-only 数据点。

## 核心组件审计

---

### StatMiniCard（`components/ui/StatMiniCard.tsx`）

**问题**：`sub` 参数存在但**无强制要求**，调用方几乎都不传 `sub`，导致 L1 副标题为空。
**影响范围**：全站所有使用 StatMiniCard 的位置（20+ 处）

**建议修改**：
- 将 `sub` 改为**强制参数**（或在 eslint-plugin 中标记 required）
- 内置 `sub` 内容公式：`{指标名}：{定义}，健康范围 {range}`

**当前**：
```tsx
{sub && <p className="text-xs text-[var(--text-muted)] mt-0.5">{sub}</p>}
```
**建议**：
```tsx
<p className="text-xs text-[var(--text-muted)] mt-0.5">{sub || '暂无说明'}</p>
// 同时在 props 中加 sub?: string (带 eslint warning when empty for known metrics)
```

---

### RateCard（`components/ui/RateCard.tsx`）

**问题**：`sub` 参数存在，但大量调用未传，百分比大数字无任何上下文。目标线显示为"目标 N%"但不解释该目标从哪来。

**建议**：
- 对所有效率类指标（参与率/打卡率/触达率/转化率）的 RateCard 强制传 `sub` 说明"该指标是什么 + 目标来源（如：设置中配置的月度目标）"

---

### StatCard（`components/shared/StatCard.tsx`）

**问题**：`target` 显示为"目标 N"，无单位无来源说明。达成率颜色（绿/黄/红）阈值 80% 和 100% 未向用户展示。

**建议**：
- 在进度条下方加一行 `<10px muted>≥100% 达标 | ≥80% 追进 | <80% 落后</10px>`
- 或将颜色阈值放入 tooltip on hover 达成率数字

---

### BrandDot（`components/ui/BrandDot.tsx`）

**状态**：组件已实现，CSS 规范完整，**但全站 0 次调用**。

**影响**：L1 密集布局方案（表格/图表）完全缺失。设计体系中最重要的 L1 工具形同虚设。

**全局优先行动**：在所有表格表头添加 BrandDot，逐列补充 tooltip。
