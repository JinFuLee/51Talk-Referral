# UX 审计报告 — 转介绍运营专员视角

> 审计日期：2026-03-25
> 审计方法：角色模拟 — 以 51Talk 泰国转介绍运营专员身份逐页体验
> 覆盖页面：7 个高频页面（打卡管理 / 首页 Dashboard / 高潜学员 / 触达监控 / 学员明细 / 漏斗分析 / 团队汇总）

---

## 角色背景

**我是谁**：51Talk 泰国转介绍运营专员
**我的日常工作**：
- 每天上午 9:00 检查昨天数据，找出执行缺口
- 找未打卡学员 → 分配给 CC 跟进
- 监控每个 CC 的触达率 / 参与率
- 跟踪高潜学员的转化进展
- 每周整理数据给总监汇报

## 页面 1：打卡管理（/checkin）

### 我的使用场景
- **每天必用**：早上第一个打开，了解昨天打卡率，确认是否有 CC 执行不力
- **下午复核**：跟进昨天分配的未打卡学员是否被联系
- **月末汇报**：统计当月打卡趋势，给总监看改善情况

### 当前体验（3/5 分）

**信息找得到吗？**
- 汇总视图（CC/SS/LP 三栏）能看到打卡率总体情况，信息密度适中
- 颜色编码（绿/橙/红）直觉清晰
- "未打卡跟进" Tab 是我最需要的，但藏在第 4 个 Tab，每次都要多点一下

**操作顺畅吗？**
- Tab 切换逻辑清晰
- 未打卡跟进 Tab 有围场筛选 + 团队筛选，基本够用
- **致命缺陷**：未打卡学员列表没有"批量导出"功能，我每天需要把这份名单发给 CC，现在只能截图或手动整理

**多余的步骤？**
- 打开页面后默认是"汇总视图"，但我 90% 的时间需要的是"未打卡跟进"，每次都要点 3 个 Tab 才到目的地（汇总 → 排行 → 团队明细 → 未打卡跟进）

### 痛点

| 痛点 | 描述 | 影响我的工作效率 |
|------|------|----------------|
| 缺导出功能 | 未打卡跟进列表没有 CSV/Excel 导出 | 每天需要手动整理名单发 CC，约浪费 20-30 分钟 |
| 默认 Tab 错误 | 打开默认显示"汇总视图"，我的高频操作是"未打卡跟进" | 每次操作多 3 次点击，累计每天约 20 次无效点击 |
| 无排序功能 | 未打卡跟进列表按质量评分排序，但我需要按"卡到期天数"排序来识别最紧急的学员 | 紧急学员可能被埋在中间，影响跟进优先级 |
| 缺"上次跟进日期"列 | 表格有 CC 末次联系日期，但没有"这次分配时间"字段 | 无法判断今天分配的任务是否被 CC 及时处理 |
| 无打卡趋势图 | 看到今天打卡率是 40%，但不知道昨天是 35% 还是 60% | 无法判断是在好转还是恶化 |

### 我希望的体验

```
打开打卡管理 →
  顶部：今日打卡率大数字（绿/橙/红）+ 7 天趋势迷你图（一眼知道在变好还是变坏）
  默认展示：未打卡跟进列表（我的核心需求）
  右上角：[导出名单] 按钮 → 一键生成 Excel，包含学员ID/围场/负责CC/质量评分/卡到期
  表格有列排序：卡到期天数/质量评分/末次联系日期
```

### 改善建议（具体到组件级）

| 建议 | Before | After | 优先级 |
|------|--------|-------|--------|
| 默认 Tab 改为"未打卡跟进" | 进入页面 4 次点击才到跟进列表 | 进入页面直接看到跟进列表 | P0 |
| 增加 CSV 导出按钮 | 无导出，手动整理名单 | 点击导出，获得包含联系信息的 Excel | P0 |
| 表头增加列排序功能 | 固定按质量评分排序 | 可按卡到期天数/末次联系/评分排序 | P1 |
| 顶部加打卡率趋势迷你图（7天） | 只有今天的打卡率 | 看到 7 天折线，一眼判断趋势 | P1 |
| 增加"跟进状态"列 | 无法追踪分配后是否被处理 | 显示"已联系/未联系"，可由 CC 回填 | P2 |

## 页面 2：首页 Dashboard（/）

### 我的使用场景
- **每天早上**：快速确认整体 KPI 达成情况，判断今天重点推什么
- **总监来问**：立刻能拉出关键数字汇报

### 当前体验（4/5 分）

**信息找得到吗？**
- KPI 8 项全维度卡片非常完整（当前值/目标/差距/日均/时间进度差），对分析师很有用
- 时间进度条逻辑清晰，"追进度需日均" 这个维度特别有价值
- D2b 全站基准区域的数据说明很贴心（每个指标都有副标题解释）

**操作顺畅吗？**
- 加载后整页信息量很大，需要滚动很多才能看完
- 岗位视角筛选器（全部/CC/SS/LP）很好用，切换后 KPI 卡片会重新过滤

**多余的步骤？**
- KPI 8 项卡片和顶部的 StatCard 有信息重叠，同一个指标在不同卡片展示了两次

### 痛点

| 痛点 | 描述 | 影响我的工作效率 |
|------|------|----------------|
| 关键信号埋得太深 | 今天最需要关注的是哪个指标？页面没有给出结论，需要我自己看所有数字再判断 | 每天 10-15 分钟阅读页面，高认知负担 |
| 缺"今日重点行动"区 | 页面是数据展示，不是行动指引；我需要知道"今天该让 CC 重点推什么" | 还是需要自己做决策，没有系统支持 |
| 漏斗图无时间筛选 | 只有本月累计数据，无法看昨天或上周的漏斗表现 | 无法判断是今天的问题还是月初就开始落后 |
| 月度达成环形进度和 KPI8 卡片重叠 | 单量达成率在环形图和 KPI8 卡片都有，内容重复 | 阅读负担增加 |

### 我希望的体验

```
首页顶部 = "今日运营摘要"（3 行以内）：
  🔴 触达率 32%（低于目标，需要 CC-A / CC-B 今天重点跟进）
  🟡 打卡率 38%（接近目标，继续维持）
  🟢 注册 → 预约转化率 76%（达标）
然后才是详细数字卡片
```

### 改善建议（具体到组件级）

| 建议 | Before | After | 优先级 |
|------|--------|-------|--------|
| 增加顶部"今日运营摘要"区 | 需要自己读完所有卡片才能总结情况 | 系统自动生成 3 条重点行动信号 | P1 |
| 合并重复展示 | KPI8 卡片 + StatCard 重复展示同一指标 | KPI8 卡片为主，StatCard 下架 | P1 |
| 漏斗图增加时间筛选（本月/上周/昨天） | 仅本月累计 | 可切换时间维度，快速定位问题发生时间段 | P2 |

## 页面 3：高潜学员（/high-potential）

### 我的使用场景
- **每天跟踪**：核实高潜学员是否被 CC 及时跟进
- **发现遗漏**：找到失联 >7 天但仍有高潜力的学员，催 CC 联系

### 当前体验（3/5 分）

**信息找得到吗？**
- 卡片设计信息密度合理（带新/出席/付费 + 失联天数 + urgency 颜色标记）
- Warroom 数据（打卡次数/窗口期/最后接通）集成得很好
- 但是：卡片数量多的时候（>20 人），我找不到按 CC 筛选的入口

**操作顺畅吗？**
- 卡片布局是网格，适合快速浏览
- 但无法按"CC 姓名"或"围场"筛选
- 无法按"失联天数"排序（最紧急的应该在最上面）

**多余的步骤？**
- 想找某个 CC 负责的学员，现在只能逐一看卡片标注的 CC 姓名

### 痛点

| 痛点 | 描述 | 影响我的工作效率 |
|------|------|----------------|
| 无 CC 筛选 | 无法按照 CC 姓名过滤，看不到"某 CC 负责的所有高潜学员" | 管理 CC 跟进情况时，每次需要逐一对照 |
| 无排序控件 | 无法按失联天数/urgency/围场排序 | 紧急学员可能在第 3 页，无法优先处理 |
| 无跟进状态追踪 | 只知道"失联 N 天"，不知道是否已分配给 CC 处理 | 容易重复分配或漏分配 |
| 数量多时浏览效率低 | 50+ 学员时，卡片视图需要大量滚动 | 切换列表视图会更高效，但没有视图切换 |

### 我希望的体验

```
高潜学员页面顶部：
  [CC 筛选下拉] [围场筛选] [urgency 筛选: 全部/紧急/关注/稳定] [排序: 失联天数↓]
  [切换: 卡片视图 | 列表视图]

列表视图 = 表格，列：学员ID / 围场 / CC负责人 / 带新数 / 失联天数 / 窗口期 / urgency / 操作
```

### 改善建议（具体到组件级）

| 建议 | Before | After | 优先级 |
|------|--------|-------|--------|
| 增加 CC 筛选器 | 无法按 CC 过滤高潜学员 | dropdown 选 CC 姓名，只显示该 CC 负责的高潜学员 | P0 |
| 增加失联天数排序 | 卡片固定顺序（按 student ID） | 按失联天数降序排列，最紧急的在最上面 | P0 |
| 增加列表视图切换 | 只有卡片视图，50+ 条目滚动费时 | 增加表格视图，每行 1 个学员，可排序 | P1 |
| 增加 urgency 筛选 | 无法单独查看"紧急"学员 | 筛选按钮：全部/紧急/关注/稳定 | P1 |

## 页面 4：日常触达监控（/daily-monitor）

### 我的使用场景
- **每天早上**：检查昨天 CC/SS/LP 的触达率，发现执行不力的 CC
- **异常处理**：某 CC 触达率突然下降，找到具体是谁、下降多少

### 当前体验（4/5 分）

**信息找得到吗？**
- 三角色触达率大数字（仪表盘样式）非常直观
- 围场段触达率堆叠柱图非常有用，一眼看出哪个围场被忽视
- CC 接通排行支持 Tab 切换（CC/SS/LP），逻辑清晰

**操作顺畅吗？**
- CC 接通排行的排序是固定的，但表头显示有数据后就能看排名，基本满足需求
- 触达率 × 转化率散点图是分析维度，实际工作中不常用，但占了很大篇幅

**多余的步骤？**
- 触达率仪表盘 + 角色对比卡 + 漏斗 三栏并排展示，信息比较分散

### 痛点

| 痛点 | 描述 | 影响我的工作效率 |
|------|------|----------------|
| 排行表缺"目标达成"列 | CC 排行只显示触达率数字，无法直接看出是否达标 | 需要自己心算和记忆目标值才能判断 |
| 无"环比变化"列 | 无法看到"这个 CC 的触达率比昨天高了还是低了" | 无法判断趋势，只能看今天快照 |
| 排行表无法导出 | 无法把今天的排行发给组长/总监 | 需要截图，格式不整洁 |
| 散点图和我的工作相关性低 | 触达 × 转化散点图是分析工具，不是运营操作工具 | 占了 1/3 页面，分散注意力 |

### 我希望的体验

```
CC 接通排行表新增：
  - "目标" 列（从设置读取）
  - "达成" 列（实际/目标，颜色标红绿）
  - "昨日变化" 列（+3.2pp 绿 / -1.5pp 红）
  - [导出本日排行] 按钮
```

### 改善建议（具体到组件级）

| 建议 | Before | After | 优先级 |
|------|--------|-------|--------|
| 排行表增加目标/达成列 | 只显示触达率数字 | 增加目标值和达成率，配色标红绿 | P0 |
| 排行表增加环比变化列 | 只有当期快照 | 显示较昨日变化（+/-N pp），颜色编码 | P1 |
| 增加排行导出功能 | 无导出 | 点击导出，获得今日触达率排行 Excel | P1 |
| 散点图折叠默认收起 | 散点图默认展开，占大量空间 | 默认收起，点击展开（分析师才需要） | P2 |

## 页面 5：学员明细（/members）

### 我的使用场景
- **按需查询**：知道某个学员 ID，查他的详细情况
- **排查问题**：找到某个 CC 下，15 天+未联系的高风险学员
- **验证猜想**：围场 0-30 天的学员是否真的打卡率更高？

### 当前体验（4/5 分）

**信息找得到吗？**
- 筛选器（围场/CC/失联天数/次卡健康度/有带新记录）覆盖了我主要的查询需求
- 点击行展开详情抽屉（59 字段）非常实用，不需要跳转页面
- 表格列顺序合理（ID / 围场 / 生命周期 / CC / 打卡天 / 课耗 / 转码 / 奖励状态 / 卡到期 / 末次拨打）

**操作顺畅吗？**
- 围场筛选是文字输入（如"0-30"），不是下拉，需要记住格式
- 每页 20 条，总数很多时翻页很慢
- 没有"导出当前筛选结果"功能

**多余的步骤？**
- 找到某个 CC 下的"高风险失联学员"需要组合两个筛选条件，但 CC 筛选是文字输入，容易输错

### 痛点

| 痛点 | 描述 | 影响我的工作效率 |
|------|------|----------------|
| 围场筛选是文字输入 | 需要记住"0-30"、"31-60"等格式，输入错误时无提示 | 低频操作容易出错，用户体验差 |
| 无导出功能 | 无法导出当前筛选结果 | 需要截图整理名单，浪费时间 |
| 无批量操作 | 选中多行后无法批量分配/标记 | 每次只能单一处理学员，效率低 |
| 分页翻页效率低 | 每页 20 条，无法调整每页大小 | 想看更多时需要多次翻页 |

### 我希望的体验

```
围场筛选改为下拉多选（0-30 / 31-60 / 61-90 / 91-180 / 181+）
导出按钮放在列表卡片标题右侧
每页大小选择：20 / 50 / 100
```

### 改善建议（具体到组件级）

| 建议 | Before | After | 优先级 |
|------|--------|-------|--------|
| 围场筛选改为下拉选择 | 文字输入框，需记忆格式 | 下拉多选（0-30/31-60/61-90/91-180/181+） | P1 |
| 增加导出功能 | 无法导出筛选结果 | [导出 CSV] 按钮，输出当前筛选+排序的全量数据 | P1 |
| 每页大小可调 | 固定 20 条/页 | 增加 50/100 条/页选项 | P2 |

## 页面 6：漏斗分析（/funnel）

### 我的使用场景
- **每周汇报**：给总监展示本月漏斗各环节达成情况
- **找问题节点**：哪个转化率最低，是导致付费数不足的根因

### 当前体验（4/5 分）

**信息找得到吗？**
- 完整邀约漏斗（注册→邀约→出席→付费）结构完整
- 场景推演表格（"如果出席付费率提升到 30% 能多多少付费"）非常有价值，汇报时能直接拿来用
- 转化率柱状图颜色标红绿，直观

**操作顺畅吗？**
- 页面加载时间比其他页面稍慢（多个 SWR 请求）
- 没有时间维度筛选（本月/上月/本周）

**多余的步骤？**
- 漏斗环节表格和完整邀约漏斗两个卡片有功能重叠（都展示了注册/出席/付费环节）

### 痛点

| 痛点 | 描述 | 影响我的工作效率 |
|------|------|----------------|
| 缺时间维度筛选 | 只有本月累计，无法对比上周 vs 本周表现 | 汇报时无法展示改善趋势 |
| 无口径筛选 | 漏斗是合并所有口径，无法单独看 CC/SS/LP 的漏斗 | 无法定位具体哪个岗位的哪个转化环节是问题 |
| 场景推演无法修改参数 | 场景转化率是系统预设，我无法输入"如果出席率提到 35%"来看效果 | 汇报准备时需要运营用不同假设，目前无法操作 |

### 我希望的体验

```
漏斗分析页面：
  顶部筛选：[本月 ▼] [全部口径 ▼（CC/SS/LP）]
  场景推演：转化率目标值可以手动输入，实时计算影响
```

### 改善建议（具体到组件级）

| 建议 | Before | After | 优先级 |
|------|--------|-------|--------|
| 场景推演允许手动输入目标率 | 目标率系统预设，无法修改 | 输入框允许调整，实时重算影响付费数和业绩 | P1 |
| 增加岗位口径筛选（CC/SS/LP） | 全部合并展示 | 筛选后只显示对应岗位的漏斗数据 | P1 |
| 合并两个漏斗表格卡片 | 两个卡片内容重叠 | 合并为单一表格（含邀约节点），减少重复 | P2 |

## 页面 7：团队汇总（/team）

### 我的使用场景
- **每周汇报**：整理 CC/SS/LP 三岗绩效，给总监做周报
- **横向对比**：找出拖后腿的团队或个人，制定针对性行动

### 当前体验（3/5 分）

**信息找得到吗？**
- CC Tab：TeamSummaryCard 显示每个 CC 的学员数/参与率/注册/付费/业绩，信息完整
- SS/LP Tab：排名表格含参与率/打卡率/注册/付费/业绩，维度齐全
- 但 CC Tab 是卡片视图，SS/LP 是表格视图，风格不一致

**操作顺畅吗？**
- CC Tab 的卡片视图浏览效率低，CC 数量多（>10 人）时需要大量滚动
- 没有排序功能（无法按注册数/付费率/业绩排序）
- 没有导出功能

**多余的步骤？**
- 想看 CC 排名需要跳转到日常触达监控页面的"接通排行" Tab，路径不直观

### 痛点

## 全局发现（跨页面共性问题）

### 缺少"导出"能力（最高频痛点）

所有数据页面均无导出功能。运营专员每天需要：
- 将未打卡名单发给 CC（打卡管理）
- 将今日触达率排行发给组长（触达监控）
- 将高潜跟进任务分配给 CC（高潜学员）
- 准备周报数据（团队汇总）

**Before**：每天 4-5 个页面需要手动截图或复制粘贴整理，约 45-60 分钟/天
**After**：每个表格页面增加 [导出 CSV] 按钮，约 5 分钟/天
**ROI**：每天节省 40-55 分钟操作时间，减少 90% 数据整理错误

### 缺少"趋势"信号（分析维度不足）

所有页面展示的都是当期快照（本月累计或昨天），没有趋势信号：
- 打卡率今天是 40%，比昨天好还是差？
- 某 CC 的触达率这周是在提升还是下降？

建议每个关键指标旁加 7 天迷你折线图（sparkline），零改动数据源，仅前端渲染。

### 视图切换缺失（固定布局）

- 高潜学员：只有卡片视图，人多时效率低
- 团队汇总 CC Tab：只有卡片视图，无法排序
- 建议：卡片视图适合浏览（<10 条），表格视图适合操作（>10 条），提供切换按钮

### 默认状态与使用场景不匹配

- 打卡管理：默认"汇总视图"，但高频操作是"未打卡跟进"（第 4 个 Tab）
- 建议：系统可记住用户上次访问的 Tab，或根据角色设置默认 Tab
