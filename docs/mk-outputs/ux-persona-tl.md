# UX 审计报告：业务 TL（团队负责人）视角

**角色**：51Talk 泰国 CC 团队 TL，如 TH-CC05Team 负责人，管辖 5-8 名 CC
**审计时间**：2026-03-25
**审计方法**：源码逐页阅读 + 角色工作流模拟
**审计范围**：打卡管理 / 团队汇总 / 人员战力图 / 高潜学员 / 触达监控 / 围场分析

---

## 评分模板说明

每个页面按以下 5 维度评分（满分各 20 分，总分 100）：

| 维度 | 说明 |
|------|------|
| 任务匹配度 | 该页面是否支撑 TL 的核心日常任务 |
| 信息密度 | 关键信息是否"一眼可见"，无需深挖 |
| 过滤/下钻能力 | 能否按"我的团队"快速筛选 |
| 行动引导 | 看到数据后，系统是否告诉 TL 该做什么 |
| 辅导支撑 | 能否生成"CC 对比"视图帮助 TL 辅导 |

## 页面 1：打卡管理（`/checkin`）

> **TL 日常场景**：每天早上 8:00 打开，确认自己团队昨天打卡率，找到没打卡的学员，分给 CC 去跟进

### 现有功能

- **汇总视图（Tab 1）**：按 CC/SS/LP 三列展示打卡率，每列含"按团队"和"按围场"两个子表
- **打卡排行（Tab 2）**：小组排行 / 个人排行双视图，颜色编码（绿≥50% / 橙30-50% / 红<30%）
- **团队明细（Tab 3）**：`TeamDetailTab` 组件，按角色和围场展开
- **未打卡跟进（Tab 4）**：过滤器（角色 + 团队 + 销售搜索 + 围场多选），列出未打卡学员 + 质量评分 + CC末次联系时间

### TL 视角评分

| 维度 | 得分 | 评注 |
|------|------|------|
| 任务匹配度 | 18/20 | 4 个 Tab 覆盖 TL 早晨全工作流：看率 → 找差 → 确认跟进人 |
| 信息密度 | 14/20 | **问题**：汇总视图是"全团队"视角，TL 需要自己在"按团队"子表里找到自己的团队行，缺少"只看我的团队"快速入口 |
| 过滤/下钻能力 | 15/20 | 未打卡跟进 Tab 有团队过滤器，体验良好。但汇总 Tab 无快速团队切换，需先切换到 Tab 4 |
| 行动引导 | 12/20 | **缺口**：FollowupTab 展示了"超7天失联"高亮，但没有"一键生成 CC 任务列表"或"复制跟进名单"按钮 |
| 辅导支撑 | 8/20 | **缺口**：没有"哪个 CC 的打卡学员最少"的对比视图；RankingTab 是"所有人排名"，TL 找自己人需要逐行查 |

**总分：67/100**

### 关键 UX 痛点

**痛点 1：TL 无法"只看我的团队"**
汇总视图（Tab 1）展示的是 CC/SS/LP 全渠道汇总，`by_team` 子表虽然有团队粒度数据，但 TL 需要在 4-8 行里自己找自己的团队。打卡排行（Tab 2）的个人排行混合了所有 CC，TL 管的 5-8 人淹没在整体名单里。
**缺少的功能**：顶部全局团队切换器，或"我的团队"快速预设。

**痛点 2：未打卡学员的行动路径不完整**
Tab 4（未打卡跟进）能找到学员 + 负责人，但展示的是"系统数据展示"而非"执行入口"。TL 看到"CC 小明负责的 3 名学员超 7 天失联"后，系统没有提供任何后续动作（一键导出名单/生成钉钉消息/标记跟进状态）。
**缺少的功能**：行级"标记已跟进" + 批量导出功能。

**痛点 3：昨日 vs 今日数据无时间标注**
页面没有显示"数据截止时间"，TL 不知道看到的打卡率是截止今天还是昨天。这在每日早晨查看时会产生困惑。
**缺少的功能**：页头数据时效标注（"数据截至 YYYY-MM-DD 昨日收盘"）。

## 页面 2：团队汇总（`/team`）

> **TL 日常场景**：看自己团队在所有 CC 团队中排第几，哪个指标拖了后腿

### 现有功能

- **CC Tab**：`TeamSummaryCard` 卡片网格，每张卡片显示：团队名 / 有效学员 / 参与率 / 注册数 / 付费数 / 打卡率 / CC触达率 / 业绩(USD)
- **SS/LP Tab**：排名表格（排名 + 姓名 + 组名 + 学员数 + 参与率 + 打卡率 + 注册数 + 付费数 + 业绩）
- **柱状图**：注册 vs 付费对比

### TL 视角评分

| 维度 | 得分 | 评注 |
|------|------|------|
| 任务匹配度 | 12/20 | **问题**：CC Tab 展示的是"每个CC个人"的卡片（cc_name = 个人名），而非"每个团队"的汇总。TL 想看"TH-CC05Team vs TH-CC03Team"，但看到的是一个个 CC 个人卡片 |
| 信息密度 | 13/20 | 卡片信息密度尚可，但缺少"排名"字段，TL 不知道自己团队在全体中的位置 |
| 过滤/下钻能力 | 8/20 | **严重缺口**：无任何团队过滤器。TL 找自己团队的 CC 需要眼扫整页卡片网格 |
| 行动引导 | 6/20 | **严重缺口**：页面是"数据展示"，无任何指示"这个指标落后了，建议做 X" |
| 辅导支撑 | 10/20 | 卡片包含参与率/打卡率/触达率，可作横向比较，但没有排名徽章和红/黄/绿色编码 |

**总分：49/100**

### 关键 UX 痛点

**痛点 1：页面名称与实际内容错位**
页面标题叫"团队汇总"，但 CC Tab 的数据粒度是"个人（cc_name）"而非"团队（cc_group）"。TL 期望看到"团队 A vs 团队 B"的比较，实际看到的是"张三 vs 李四"。
**建议**：CC Tab 增加两个子视图 — "团队视图"（按 cc_group 聚合）和"个人视图"（现有）。

**痛点 2：卡片没有排名，无法判断自己的相对位置**
`TeamSummaryCard` 组件展示了绝对数值（参与率 X%、注册 N 条），但没有相对排名。TL 面对 8-12 张卡片，无法一眼判断"我的人在全体里排第几"。
**建议**：卡片左上角增加排名徽章（金/银/铜 + 数字），对应 Team 页面 SS/LP Tab 的 `RankBadge` 组件（已存在，复用即可）。

**痛点 3：无"与目标对比"信息**
卡片显示的是本月累计数，无法判断是否跟上进度。TL 看到"注册 23 条"不知道目标是 30 还是 15。
**建议**：参考 CLAUDE.md "双差额体系"，卡片增加目标差（实际 vs 目标）小徽章。

## 页面 3：人员战力图（`/personnel-matrix`）

> **TL 日常场景**：某个 CC 带新数高但出席率低，系统能帮我定位这个问题吗？找出谁是"质量低效"的 CC

### 现有功能

- **CC Tab**：
  - 热力矩阵（CC × 围场段，着色维度：带新系数/参与率/打卡率/触达率）
  - 带新系数 × 付费金额四象限散点图
  - 点击 CC → 雷达图弹层（多维能力展示）
  - 点击单元格 → 学员明细下钻（stdt_id + 姓名 + 付费金额）
- **SS/LP Tab**：按注册数排序的全量个人表格

### TL 视角评分

| 维度 | 得分 | 评注 |
|------|------|------|
| 任务匹配度 | 17/20 | 热力矩阵是 TL 找"异常 CC"的最强工具，维度可切换，覆盖 TL 核心需求 |
| 信息密度 | 16/20 | 矩阵 + 散点图 + 雷达图三层递进，信息密度优秀 |
| 过滤/下钻能力 | 15/20 | 可点击 CC 看雷达图，可点击单元格下钻学员。缺少"只显示我的团队的 CC" |
| 行动引导 | 10/20 | **缺口**：四象限散点图（带新高/带新低 × 付费高/付费低）有四个象限，但没有文字说明各象限的含义和 TL 应采取的行动 |
| 辅导支撑 | 17/20 | 雷达图是辅导利器，可以直观展示 CC 多维能力。如果能"选两个 CC 一起显示"会更强 |

**总分：75/100**

### 关键 UX 痛点

**痛点 1：热力矩阵行排序不支持 TL 团队预筛选**
CC Tab 展示所有 CC 的热力矩阵，TL 管的 5-8 人混在 20-30 行里。目前无法"只看 TH-CC05Team 的 CC"。
**建议**：矩阵顶部增加"团队"快速过滤器（下拉选择 cc_group），过滤后矩阵只展示该团队 CC，行数大幅减少，更聚焦。

**痛点 2：散点图四象限无文字标注**
`EfficiencyScatter` 的 X 轴是带新系数、Y 轴是付费金额，但四个象限没有标注（如"高效 / 低效 / 潜力 / 拖累"）。TL 需要自己理解坐标含义，认知负荷高。
**建议**：四象限各加一行文字标注，如右上"明星（高带新×高业绩）"、左下"待激活"。

**痛点 3：雷达图仅支持单 CC 查看**
点击 CC 可看该 CC 的雷达图，但 TL 辅导时最需要"与 top performer 对比"。雷达图只有一个人的数据，无法做"差距可视化"。
**建议**：雷达图支持同时叠加 2-3 个 CC（或固定叠加"团队平均线"），实现对比辅导。

## 页面 4：高潜学员（`/high-potential`）

> **TL 日常场景**：我团队负责的高潜学员有哪些？能按我的团队筛选吗？

### 现有功能

- 卡片网格展示高潜学员，每张卡片包含：
  - 学员 ID / 围场段 / 紧急程度（红/黄/绿边框）
  - 带新数 / 出席数 / 付费数
  - 负责的 CC / SS / LP 姓名和组别
  - 参与深度（深度/浅度）/ 失联天数
  - Warroom 数据：打卡次数(7d) / 窗口期 / 最后接通日期

### TL 视角评分

| 维度 | 得分 | 评注 |
|------|------|------|
| 任务匹配度 | 14/20 | 卡片信息覆盖了跟进所需的关键字段，但缺少"按 TL 团队筛选"能力 |
| 信息密度 | 16/20 | 卡片信息层次清晰，紧急程度颜色编码直观 |
| 过滤/下钻能力 | 5/20 | **严重缺口**：整页无任何筛选器。TL 无法按"我的团队的 CC"过滤。所有高潜学员全部展示，TL 管的可能只有 5-10 张，但页面可能有几十张卡片 |
| 行动引导 | 10/20 | 失联天数 > 14 天会红色高亮，有紧急程度标签。但没有"下一步行动"的操作按钮 |
| 辅导支撑 | 8/20 | 卡片有 CC 姓名，TL 能看到"这个高潜学员属于我的哪个 CC"，但无法聚合"某 CC 的高潜学员整体状况" |

**总分：53/100**

### 关键 UX 痛点

**痛点 1（最高优先级）：无任何筛选器**
高潜学员页面是"全公司所有高潜学员"的视图，完全没有过滤器。TL 打开这个页面，面对的是几十张混杂的卡片，无法快速找到"我的 CC 负责的高潜学员"。
**建议**：页面顶部增加三级过滤器：
1. 岗位（CC/SS/LP）
2. 团队（cc_group，如 TH-CC05Team）
3. CC 姓名（cc_name）

**痛点 2：紧急程度标签无标准定义展示**
卡片有"紧急/关注/稳定"标签，但页面没有说明"红色=什么条件触发"。TL 看到"紧急"，不知道是因为失联天数多、窗口期快到期，还是其他原因。
**建议**：页头增加 `GlossaryBanner`（代码中已有此组件），展示紧急程度的评分规则。

**痛点 3：学员 ID 而非学员姓名**
卡片展示的是学员 ID（`#12345`），而非学员姓名。TL 辅导 CC 时说"#12345 这个学员要关注"，CC 不知道是谁。需要对照另一个系统查名字。
**建议**：卡片主标题位置增加学员姓名（若数据可用），ID 作为副标题展示。

## 页面 5：日常触达监控（`/daily-monitor`）

> **TL 日常场景**：我的 CC 触达率够吗？谁在认真打电话，谁在划水？

### 现有功能

- 顶部三个大数字 gauge：CC/SS/LP 触达率
- 角色对比图（CC vs SS vs LP 触达率对比柱状图）
- 转介绍漏斗（注册/邀约/出席/付费/业绩/打卡率）
- 围场段触达率分布堆叠柱图
- 接通排行（Tab 切换：CC/SS/LP，显示个人触达率排行）
- 触达率 × 转化率散点图

### TL 视角评分

| 维度 | 得分 | 评注 |
|------|------|------|
| 任务匹配度 | 15/20 | 接通排行（CCContactRanking）是 TL 的核心工具，能看到每个 CC 的触达率排名 |
| 信息密度 | 16/20 | 页面信息层次清晰，从宏观（三大数字）到明细（个人排行）递进 |
| 过滤/下钻能力 | 8/20 | **缺口**：接通排行只有 CC/SS/LP Tab 切换，无团队过滤。TL 管的 5-8 个 CC 混在全体名单里，无法只看自己团队 |
| 行动引导 | 10/20 | 散点图右上角是"最优区间"，但没有明确写出"触达率低于 X% 需要辅导"的红线 |
| 辅导支撑 | 12/20 | 个人排行可以看 CC 触达数，但缺少"目标完成率"对比（不知道每个 CC 今天应该打多少电话） |

**总分：61/100**

### 关键 UX 痛点

**痛点 1：接通排行无团队过滤**
CCContactRanking 组件展示全部 CC 的排行，无法只显示 TH-CC05Team 的 5-8 人。TL 需要自己在名单里找到自己的 CC，效率低。
**建议**：排行列表顶部增加"团队"下拉过滤器，切换后只展示该团队成员。

**痛点 2：漏斗数据是全局口径，TL 无法拆解"我的团队贡献了多少"**
转介绍漏斗显示的是全公司的注册/邀约/出席/付费，TL 看到"付费 23 单"无法知道自己团队贡献了多少。
**建议**：漏斗卡片增加团队筛选维度，或在漏斗下方增加"按团队分解"的柱状图。

**痛点 3：散点图的"我的人在哪里"不明显**
触达 × 转化散点图（ContactConversionScatter）中，TL 无法识别哪些点是自己团队的 CC。
**建议**：散点图支持"高亮我的团队"——通过颜色区分，让 TL 的团队成员以不同颜色展示。

## 页面 6：围场分析（`/enclosure`）

> **TL 日常场景**：我团队的学员围场分布是什么？0-30 天新客多还是 91+ 老客多？

### 现有功能（基于 enclosure/page.tsx 结构）

- 全部汇总 / CC / SS / LP 四个 Tab
- CC Tab：按 CC 个人分组的围场指标表格（CC名 / 组别 / 围场段 / 学员数 / 参与率 / 打卡率 / 触达率 / 注册数 / 付费数 / 业绩）
- 围场段过滤器（0-30 / 31-60 / 61-90 / 91-180 / 181+）
- 柱状图：按围场段的注册 vs 付费对比
- 围场热力图组件（`EnclosureHeatmap`）
- 围场雷达图（`MetricRadar`）

### TL 视角评分

| 维度 | 得分 | 评注 |
|------|------|------|
| 任务匹配度 | 14/20 | 围场段过滤器满足"看特定围场学员"需求，但缺少"我的团队在各围场的分布概览" |
| 信息密度 | 14/20 | 表格信息完整，但"全公司所有 CC × 所有围场"数据量大，TL 找自己团队的行困难 |
| 过滤/下钻能力 | 12/20 | 有围场段过滤器，但无"团队"过滤器 |
| 行动引导 | 8/20 | **缺口**：TL 看到"0-30 天新客 30 人、91+ 老客 50 人"，页面没有给出"老客比例高意味着什么" |
| 辅导支撑 | 10/20 | 可以看每个 CC 的各围场指标，辅导时可用，但仍需先过滤自己团队的 CC |

**总分：58/100**

### 关键 UX 痛点

**痛点 1：表格是全员视图，TL 找自己团队效率低**
CC 围场表格展示所有 CC 的所有围场数据，行数可能超过 50 行（10 名 CC × 5 个围场段）。TL 每次需要手动扫描找到自己的 5-8 名 CC。
**建议**：增加"团队"列过滤器（cc_group 下拉），选择后只展示该团队的 CC 行。

**痛点 2：缺少"团队围场分布概览"卡片**
TL 想知道"我的团队整体上 0-30 天新客占多少、91+ 老客占多少"，这是制定本月策略的基础（新客多 → 重打卡；老客多 → 重维护）。但现在没有这个汇总视图。
**建议**：在 CC Tab 顶部增加一个"我的团队围场分布"饼图或分段条形图（需先有团队过滤）。

## 跨页面系统性问题

### 问题 A（最高优先级）：缺乏"我的团队"全局过滤层

**所有 6 个页面都缺少"按 TL 团队过滤"的能力。** TL 每次打开任何页面，都是看全公司数据，需要自己扫描定位自己的 5-8 人。这是 TL 用户体验的最核心缺口。

| 页面 | 当前状态 | 需要增加 |
|------|---------|---------|
| 打卡管理 | 无团队过滤 | 顶部团队选择器 + Tab 4 团队默认值 |
| 团队汇总 | 无团队过滤 | CC Tab 增加 cc_group 过滤 |
| 人员战力图 | 无团队过滤 | 矩阵行过滤：只显示选定团队的 CC |
| 高潜学员 | 无任何过滤器 | 完整过滤器体系（CC/团队/CC名） |
| 触达监控 | 无团队过滤 | 接通排行增加团队过滤 |
| 围场分析 | 无团队过滤 | 表格增加 cc_group 列过滤 |

**根本设计方案**：在 `Topbar` 或 `Sidebar` 增加全局"我的团队"切换器（或个人偏好设置），选择后持久化到 localStorage，所有页面自动应用此过滤维度。

### 问题 B：缺少"行动建议"层

现有系统是优秀的"数据展示"工具，但对 TL 而言缺少"行动建议"层：
- 看到打卡率红色（<30%）→ 没有"建议 TL 做什么"的提示
- 看到某 CC 带新高但出席低 → 没有"辅导方向建议"的系统提示
- 看到高潜学员失联 14 天 → 没有"立即联系"操作按钮

**建议**：各页面增加"运营建议"轻提示组件（与已有的 `RiskAlert` 组件对齐），基于数据阈值触发。

### 问题 C：数据时效不透明

所有页面均未显示"数据截止时间"。TL 早上查看时，不确定看到的是昨天还是前天的数据，可能基于过期数据做决策。
**建议**：Header 区域增加"数据截至 YYYY-MM-DD（T-1）"全局标注，各页面继承展示。

### 问题 D：CC 辅导核心视图缺失

TL 辅导 CC 的关键需求是"让某个 CC 看到自己和 top performer 的差距"。目前：
- 人员战力图的雷达图只支持单人
- 无"CC-A vs CC-B 指标对比"视图
- 无"选定 CC 的月度趋势"视图

**建议**：人员战力图页增加"CC 对比"模块：多选 2-3 个 CC，生成雷达图叠加对比（参考 CLAUDE.md 中的 5-Why 分析方法论，从"差距"到"根因"）。

## 总结评分

| 页面 | 得分 | 核心问题 |
|------|------|---------|
| 打卡管理 | 67/100 | 缺"我的团队"过滤，未打卡行动路径不完整 |
| 团队汇总 | 49/100 | 页面名称与内容错位，缺排名和目标对比 |
| 人员战力图 | 75/100 | 最强工具，缺团队过滤和双人雷达对比 |
| 高潜学员 | 53/100 | 无任何过滤器，是最大 UX 缺口 |
| 触达监控 | 61/100 | 接通排行缺团队过滤，漏斗无团队分解 |
| 围场分析 | 58/100 | 全员表格，TL 找自己团队效率低 |
| **综合均分** | **60/100** | **系统性问题：缺"我的团队"视角** |

> **核心结论**：系统数据维度覆盖完整，是优秀的数据分析平台，但当前是"数据分析师"视角（全量、客观、多维），而非"TL 日常运营"视角（聚焦、有边界、可操作）。最高价值改进是增加"我的团队"全局过滤层，让每个页面自动聚焦 TL 管辖范围内的数据。
