## 逐页面审计

### 1. 首页 Dashboard（`/`）— 总览 Dashboard

**模拟场景**：周一开会前 5 分钟打开系统

#### 发现的问题

**严重问题（直接影响业务总监使用）：**

1. **缺乏"一句话结论"**
   - 当前状态：打开页面看到 KPI 卡片堆叠（注册 / 预约 / 出席 / 付费 / 业绩），每张卡片 8 行数据
   - 总监视角："本月能完成吗"这个问题没有被直接回答
   - 缺失内容：顶部没有"本月业绩达成率 XX%，时间进度 XX%，状态：🔴 落后 / 🟢 跑赢"的摘要横幅
   - Before：总监需要自己解读 5 张卡片 + 3 个环形图才能得出结论
   - After：1 行横幅即可，格式如"业绩达成 62%，时间进度 75%，缺口 $3,200 → 每日需 $427"

2. **KPI 8 项卡片对业务总监认知负担过重**
   - "达标需日均"/"追进度需日均"/"效率提升需求"是运营细节
   - 业务总监只需要："当前达成率 vs 时间进度" + "完成月目标还差多少"
   - 8 行数据的卡片密度是运营经理/CC 主管的视角，不是决策者视角

3. **"全站基准（D2b）"板块在总监角度意义不明**
   - "带新系数 1.8"、"带货比 0.12"——总监不知道 1.8 是好还是坏，没有行业基准对比
   - subtitle 虽然有解释（9px 灰色字），但业务总监不会主动去读那么小的文字
   - 缺失：红绿灯状态标记（好/警戒/差），让总监一眼判断

4. **"转介绍 vs 其他渠道"对比在首页完全缺失**
   - 总监视角：转介绍渠道占总销售额多少比例？今天不知道
   - 渠道对比信息在 `/channel` 页面，而非首页
   - 首页只有转介绍内部的漏斗，没有"转介绍 vs 市场渠道" ROI 横向对比

**中等问题：**

5. **时间进度信息条是好设计，但位置偏后**
   - 时间进度条在 KPI 卡片之前展示，这一点做得好
   - 但建议提到最顶部作为"决策上下文"，总监第一眼就要知道"现在月初 / 月中 / 月末"

6. **"数据源状态"板块对业务总监无意义**
   - 总监不关心"数据源是否有文件"，这是运营/技术细节
   - 建议隐藏或折叠到底部

#### 术语理解测试

| 术语 | 业务总监能理解？ | 原因 |
|------|---------------|------|
| 带新系数 | 否（需要读 9px 灰色 subtitle） | 数字 "1.8" 无上下文 |
| 带货比 | 否 | "带货"在转介绍语境有歧义 |
| 围场 | 否 | 没有解释"围场 = 付费后第几天" |
| CC/SS/LP | 需要背景知识 | 首页没有角色说明 |
| KPI 8 项中的"追进度需日均" | 否 | 需要理解"时间进度"概念 |
| 注册转化率 | 是 | 通用业务术语 |

**结论**：首页对业务总监的评分 **40/100**。数据完整性很高，但信息架构是"给运营经理看的"，不是"给总监开会用的"。

### 2. 汇报模式（`/present`）— 汇报沉浸模式

**模拟场景**：开会用，需要全屏演示

#### 发现的优点

1. **场景化设计很好** — 区分 "GM 汇报 / 运营总监汇报 / 对等会议"，这是正确的用户思维
2. **全屏模式 + 键盘导航** — 空格/方向键翻页，开会体验专业
3. **Slide 内容分工合理** — GM 场景聚焦 TargetGap + Revenue + ThreeFactorSlide，不是所有数据都塞进去

#### 发现的问题

1. **GM 汇报场景缺少"本月总结一句话 Slide"**
   - GM 场景播放列表：TargetGapSlide → RevenueContributionSlide → ThreeFactorSlide → ConversionRateSlide → ScenarioAnalysisSlide → ChannelRevenueSlide
   - 缺失：第 1 张 Slide 应该是"本月状态摘要"（达成率 / 风险等级 / 核心结论）
   - 当前第 1 张 TargetGapSlide 直接给数字，没有"结论先行"

2. **汇报启动流程缺少"记住上次设置"**
   - 对业务总监来说每次都要重新点选"GM 汇报 + 月报"
   - 总监 80% 情况都用相同配置，建议 localStorage 缓存上次选择

3. **Slide 内容没有"与上期对比"的视觉强调**
   - TargetGapSlide 只显示当月目标 vs 实际，没有"比上月好还是差"的 MoM 指示
   - 业务总监开会最常被问"趋势怎么样"，当前 Slide 回答不了

4. **"渠道业绩贡献 Slide"表格没有 ROI 列**
   - 只有"付费金额 / 占比 / 人均金额"
   - 缺少"成本 / ROI"列——总监最想看的恰恰是"哪个渠道最划算"
   - 表格格式专业，字体清晰，这是优点

5. **Slide 暂无导出功能**
   - 总监汇报完 CEO/VP 想要截图或 PDF 导出，当前系统只能截屏
   - 没有"导出本次汇报为 PDF"按钮

**结论**：汇报模式评分 **65/100**。框架设计思路对，内容完整性好，但缺少"高管视角第一感"设计（结论先行、趋势对比、ROI 突出）。

### 3. 渠道分析（`/channel`）— ROI 判断

**模拟场景**：转介绍渠道投入产出比能一眼看到吗？

#### 发现的优点

1. **Tab 设计合理**：业绩贡献 / 净拆解 / 三因素对标 / 渠道推荐者，逻辑分层清晰
2. **饼图 + 表格双视图**，适合总监（看图）和运营（看数）
3. **"三因素对标"Tab**（预约因子 × 出席因子 × 付费因子）是高质量分析，有助于定位哪个环节拖了后腿

#### 发现的问题

1. **"ROI"在这个页面根本看不到**
   - 总监第一个问题："转介绍渠道 ROI 怎么样？"
   - 当前显示：渠道付费金额 / 注册数 / 人均金额
   - 缺失：**成本列**（每个渠道花了多少激励/运营成本）→ ROI = 收入 / 成本
   - 即使 CLAUDE.md 已注明"ROI 成本框架待对接真实数据"，页面上也应该有占位符 + 说明"成本数据待接入，届时自动计算 ROI"
   - 当前状态：总监看到"人均金额"是收入侧，没有成本侧，ROI 判断无法完成

2. **"转介绍 vs 市场渠道"对比缺失**
   - 总监关心的是"转介绍渠道相比市场渠道（自然流量/广告）哪个更划算"
   - 当前渠道分析只显示转介绍内部的 CC窄/SS窄/LP窄/宽口，不含市场渠道基准
   - 缺失：即使只放一个"市场渠道客单价 $XXX vs 转介绍客单价 $XXX"对比数字也很有价值

3. **渠道 Tab 名称"净拆解"不直观**
   - "净拆解"对业务总监来说是术语
   - 建议改为"各渠道净贡献"或"渠道净收益拆解"

4. **"渠道推荐者 TOP5"是运营级信息**
   - 显示"学员 ID + 围场 + 带新付费数"是运营经理要看的
   - 业务总监不关心具体学员 ID，关心的是"哪类学员（围场段）最有价值"
   - 建议：在总监视角隐藏或替换为"围场贡献分布图"

**结论**：渠道分析评分 **55/100**。数据结构完整，但核心问题"ROI"未被回答，且视角混合了运营细节（学员 ID）。

### 4. 漏斗分析（`/funnel`）— 转化效率

**模拟场景**：本月目标完不成的风险有多大？系统告诉你了吗？

#### 发现的优点

1. **"场景推演"（If-Then 分析）是高价值功能**
   - "出席付费率提升 5% → 影响业绩 +$2,400"这类信息对总监非常有价值
   - 让总监知道"在哪里使力收益最大"

2. **带邀约节点的完整 4 段漏斗**（注册 → 邀约 → 出席 → 付费）比普通 3 段漏斗更真实

3. **红绿黄颜色编码达成率**，一眼判断哪个环节有问题

#### 发现的问题

1. **"风险告警"不够主动**
   - 当前：漏斗数据按环节显示目标 vs 实际
   - 缺失：没有"本月完成概率预测"
   - 总监视角：他想知道的是"按当前进度，这个月能完成多少百分比？"
   - 建议：漏斗顶部加一行："按当前日均速度，预计月末达成 XX%（当前时间进度 XX%）"

2. **"场景推演"表格叙事不够 executive-friendly**
   - 当前格式：环节 / 当前转化率 / 场景转化率 / 影响注册数 / 影响付费数 / 影响业绩
   - 总监想要的：**"出席付费率提升到 30% → 月末额外增加 $2,400 业绩，约占月目标缺口的 35%"**
   - 数字到位，但缺"业绩缺口覆盖比例"这一层关联

3. **漏斗没有"历史对比"**
   - 当前只显示本月目标 vs 实际，没有"比上月同期"的对比
   - 注：CLAUDE.md 技术债第 24 条已记录此问题（依赖 SQLite 历史快照数据积累）

**结论**：漏斗分析评分 **60/100**。场景推演是亮点功能，但风险告警不主动，缺乏预测性结论。

### 5. 达成归因（`/attribution`）— 贡献归因

**模拟场景**：本月业绩贡献来自哪里？可以直接汇报吗？

#### 发现的优点

1. **4 个环形图（注册/预约/出席/付费达成率）**视觉清晰，开会可直接使用
2. **贡献拆解 Tabs（围场 / CC / 渠道 / 生命周期）**4 个维度分析全面
3. **缺口模拟器（GapSimulator）**是决策支撑工具，对总监有价值

#### 发现的问题

1. **页面标题偏技术，不够 executive-friendly**
   - 当前："达成率归因分析"
   - 建议改为："本月目标完成情况"或"业绩来源分析"

2. **贡献拆解 Tab 中的"围场"对总监不友好**
   - "围场"术语没有解释，总监不知道"0-30天围场"是什么意思
   - 建议：Tab 标签旁加简短说明"围场（付费后天数段）"

3. **"业绩达成概览行"数字被降权处理，优先级颠倒**
   - 业绩达成率 / 客单价达成率 / 注册转化率 / 出席→付费率 用 12px 灰色字展示
   - 这些恰恰是总监最关心的数字，应该是最大、最显眼的
   - 当前设计把最重要的数据降权为"辅助信息行"

4. **"缺口模拟器"功能好但入口不够明显**
   - 位于页面最底部，需要滚动才能看到
   - 建议：将"月末预测：按当前速度将完成 XX%"提到页面顶部作为摘要卡

**结论**：达成归因评分 **62/100**。功能设计完整，但高管最关心的数字被降权处理，需要信息架构重排。

## 核心差距：运营工具 vs 决策支持工具

当前系统的设计假设用户是**运营经理/CC 主管**（关心执行细节、每日数字、人员排名）。业务总监的使用场景需要**决策支持工具**（关心结论、风险、ROI、趋势方向）。

**两类用户的信息需求对比：**

| 信息类型 | 运营经理（当前系统服务对象） | 业务总监（差距所在） |
|---------|--------------------------|-------------------|
| 核心问题 | "今天哪个环节落后？谁没完成？" | "这个月能完成吗？最大风险是什么？" |
| 数字粒度 | 每日/每人明细 | 月度趋势/同期对比 |
| 术语接受度 | 接受"带新系数/围场/宽口" | 需要说人话 |
| 行动触发 | 直接找具体 CC 跟进 | 决定是否调整策略/资源 |
| 时间投入 | 每天 30 分钟 | 每周 5 分钟 |

## 优先改进建议（按 ROI 排序）

### P0 — 高价值、低成本（建议优先）

**1. 首页顶部增加"决策摘要横幅"**
- Before：总监需要解读 8 张卡片才能得出结论
- After：1 行横幅："本月业绩达成 XX%，时间进度 XX%，[状态标签：超前/跟上/落后] · 月末预测完成率 XX%"
- ROI：实现成本约 0.5 天 → 业务总监使用频率从偶尔提升到每天

**2. 漏斗分析增加"月末预测完成率"**
- Before：只显示当前实际 vs 目标，总监无法预判
- After：按当前日均速度外推到月末，显示"预计完成率 XX%（高风险 / 正常 / 超额）"
- ROI：实现成本约 1 天 → 最高频被问的问题"本月能完成吗"系统直接回答

**3. 全站术语 Tooltip 统一化**
- Before：术语解释 9px 灰色字 subtitle，总监不会读
- After：所有专业术语旁加 ⓘ 悬浮提示，说人话解释（如"围场 = 学员付费后的天数段，越早期价值越高"）
- ROI：实现成本约 1 天 → 解决总监"看不懂"问题，降低汇报前解释成本

### P1 — 中价值、中成本

**4. 渠道分析增加"ROI 状态占位符"**
- Before：渠道页面只有收入侧，ROI 无法判断
- After：渠道表格增加"成本（待接入）"列 + "ROI（待接入）"列，显示"—（数据待对接）"而非缺列
- ROI：实现成本约 0.5 天 → 让总监知道"ROI 功能存在，只是数据未就绪"，避免误判

**5. 汇报模式增加"本月总结 Slide"作为第一张**
- Before：GM 汇报直接从 TargetGapSlide（数字表格）开始
- After：第 1 张 Slide = 本月状态摘要（大字体结论 + 3 个核心数字 + 风险等级）
- ROI：实现成本约 1 天 → 符合高管汇报"结论先行"原则

**6. 汇报模式记住上次场景选择**
- Before：每次进入 `/present` 都要重新选"GM 汇报 + 月报"
- After：localStorage 缓存上次选择，进入即显示"继续上次：GM 汇报 + 月报"
- ROI：实现成本约 0.5 天 → 减少总监每次使用的操作步骤

### P2 — 未来迭代

**7. 转介绍 vs 市场渠道基准对比**
- 需要对接外部市场渠道数据源，技术复杂度较高
- 但这是业务总监最核心的战略问题

**8. 汇报 PDF 导出**
- 满足"发给 CEO/VP"的汇报场景

**9. MoM 趋势 Slide**
- 依赖历史快照数据积累（CLAUDE.md 技术债第 24 条）

## 审计结论

> **当前系统是数据完整、功能扎实的运营管理工具，但对业务总监来说是"数据仓库"，不是"决策支持工具"。**

核心差距在于信息架构：系统把所有数据以平等权重展示，没有针对"决策者 5 分钟使用场景"做信息优先级设计。**决策层需要的是结论先行 → 风险高亮 → 支撑数据可下钻**，当前系统的信息流是反过来的（细节数据 → 总监自己归纳结论）。

P0 改进（决策摘要横幅 + 月末预测 + 术语说明）工作量合计约 **2.5 天**，可让业务总监的有效使用率从当前估计的 20% 提升到 60%+。
