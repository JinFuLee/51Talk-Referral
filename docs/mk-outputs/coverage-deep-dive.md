## 四、跨数据源 Join 新维度

| Join 方式 | 产出维度 | 业务场景 | 优先级 |
|-----------|---------|---------|--------|
| D4 × D3 (stdt_id) | **个人级跟进效率矩阵**：接通次数（D3）× 接通时长（D4）× 最终付费（D4）→ 个人 CC 跟进 ROI | 识别"低投入高产出"超级学员 vs "高投入零转化"沉没成本学员，优化 CC 资源分配 | **P0** |
| D4（推荐人学员ID）× D4（带新付费数） | **推荐链价值树**：以推荐人为根节点，递归展开被推荐学员的转化情况（已有 student_network，但未展示价值量化）| 识别"金字塔型"推荐者，计算每个推荐链的总 GMV 贡献 | **P0** |
| D4（CC末次接通日期+时长）× D2（CC触达率）| **CC 质量-数量矩阵**：D2 提供触达率（量），D4 提供接通时长（质）→ 4 象限（高量高质/高量低质/低量高质/低量低质）| 当前只有触达率无法判断通话质量，接通时长是质量代理变量 | **P0** |
| D3（接通明细）× D4（带新付费）| **触达时序效应分析**：从首次接通日期到首次带新付费日期的"转化时滞"；分析 CC 触达几次后带新概率最高 | 优化 CC 跟进策略，找到最优触达次数阈值 | **P1** |
| D5（高潜标签）× D4（CC带新付费数）| **高潜兑现率**：被标记为高潜的学员最终转化为付费的比率，验证高潜标签准确性 | 如果高潜标签精度低（带新付费率 < 全量均值的 1.5x），需调整高潜判定标准 | **P1** |
| D2（CC个人）× D4（CC带新付费数）| **CC 围场贡献归因**：D2 给出 CC 负责的围场+学员数（聚合），D4 给出每个学员的实际带新付费；用学员ID 打通后，可计算每个 CC 真实带来的付费数（而不是学员数×平均） | 当前 D2 只有聚合数，D4 有学员粒度；JOIN 后可提供个人级精细贡献度 | **P1** |
| D3（邀约数）× D4（本月推荐注册人数）| **激活漏斗补全**：D3 邀约次数 + D4 注册人数 = 完整的"邀约→注册→出席→付费"全链路漏斗，比当前 D1 聚合漏斗精度更高 | 当前 /funnel 用 D1 聚合数，损失了 CC 维度下钻能力 | **P1** |
| D2b（全站汇总）× D1（区域结果）| **财务 vs 运营双口径对齐**：D2b 财务模型参与率 vs D1 区域参与率（D3 来源）→ 口径差分析，识别系统性偏差 | 财务报表口径和运营报表口径的差值如果 > 5%，需要找业务原因 | **P0** |

## 五、计算派生指标

| 基础列 | 派生指标 | 公式 | 展示页面 | 优先级 |
|--------|---------|------|---------|--------|
| CC末次接通日期 + 今日 | **失联天数** | `今日 - CC末次接通日期` | expiry_alert / members / high-potential | **P0** |
| CC末次接通时长 + 总CC拨打次数 + 有效接通定义≥30s | **有效接通率** | `接通时长≥30s的接通次数 / 总CC拨打次数`（注：需确认 D4 是否存储接通次数，或只有末次接通时长；若只有末次则退化为二值指标）| cc-followup-quality | **P0** |
| CC末次备注日期 + CC末次接通日期 | **备注-接通时滞** | `备注日期 - 接通日期`（正值=接通后有跟进；负值=备注未接通；> 7 天=跟进不及时）| cc-followup-quality | **P0** |
| CC带新付费数（D4）/ CC带新人数（D4） | **CC渠道带新转化率** | `CC带新付费数 / max(CC带新人数, 1)` | channel / student_360 | **P0** |
| SS带新付费数 / SS带新人数 | **SS渠道带新转化率** | 同上 | channel | **P1** |
| LP带新付费数 / LP带新人数 | **LP渠道带新转化率** | 同上 | channel | **P1** |
| 历史转码次数 / 本月转码次数 | **参与活跃度趋势比** | `本月转码次数 / max(历史转码次数/月份数, 0.01)` — 比值 > 1 = 活跃度上升；< 0.5 = 明显衰减 | members / enclosure | **P1** |
| 当前次卡有效期天数 / 当前次卡数 | **平均每卡有效期** | 单位：天/卡；短 = 小包策略，长 = 大包策略，不同策略对应不同续费风险 | renewal_risk / expiry_alert | **P1** |
| 总次卡数 × 客单价（D1） | **生命周期价值(LTV)代理** | `总次卡数 × D1客单价` — 近似 LTV，无需真实购买金额即可分层 | members / student_360 | **P1** |
| CC带新参与数 / 学员数 | **CC渠道参与转化率（学员粒度）** | 每个学员视角：带来参与学员 / 学员总数；与围场级参与率形成个人 vs 围场基准对比 | student_360 | **P1** |
| 末次续费距今天数 × 次卡剩余张数 | **续费紧迫度综合分** | `续费天数 × (1 + 1/max(当前次卡数, 1))` — 天数长 + 次卡少 = 高紧迫度 | renewal_risk | **P1** |
| D3 首次 CC接通日期 → D4 首次带新付费日期 | **从接通到带新付费的平均天数** | 需 stdt_id JOIN，计算中位数（P50/P75/P90）| daily_monitor / 新建页面 | **P1** |
| 上月打卡天数 - 本月打卡天数 | **打卡趋势变化量** | 正值=改善；负值=退化；|Δ| > 5 = 显著变化，需跟进 | members / checkin | **P2** |
| 上月课耗 - 本月课耗 | **课耗趋势变化量** | 负值表示学习活跃度下降，可作为主动流失预测的信号 | members / renewal_risk | **P2** |

## 六、新建仪表盘方案（P0 级别）

### 6.1 CC 跟进质量仪表盘（`/cc-followup-quality`）

**数据源**：D4（CC末次备注日期/内容/接通日期/接通时长/总CC拨打次数）

**核心场景**：当前系统有触达"量"（D2 CC触达率），但无触达"质"（接通时长/备注深度）。CC 可以刷高触达次数但每次只接通 10 秒，这在现有系统中无法检测。

**关键指标**：
- 接通质量评分（接通时长 ≥ 120s = 高质量；30-119s = 低质量；< 30s = 可疑）
- 备注及时性（接通后 24h 内备注 = 及时；> 72h = 懈怠）
- 学员失联风险（末次接通距今 > 14 天 + 次卡 ≤ 1 张 = 高风险）
- CC 质量-数量 4 象限矩阵（触达量 × 接通时长）

**API**：`GET /api/analysis/cc-followup-quality`
**前端页面**：新建 `frontend/app/[locale]/cc-followup-quality/page.tsx`

### 6.2 推荐者价值贡献仪表盘（`/referral-contributor`）

**数据源**：D4（CC/SS/LP/宽口 带新人数 + 带新付费数 + 历史转码次数）

**核心场景**：当前系统没有"哪个学员是最强推荐者"的维度。D4 有精确到渠道的带新数据，可以构建个人级贡献排行榜，指导定向激励策略（哪些人值得额外投入激励资源）。

**关键指标**：
- TOP 学员多渠道带新付费排行（CC+SS+LP+宽口总计）
- 个人带新转化漏斗（带新人数 → 带新付费数 → 转化率）
- 渠道偏好分析（某学员主要通过哪个渠道推荐）
- 历史参与深度分层（历史转码次数作为"忠诚度"代理变量）

**API**：`GET /api/analysis/referral-contributor`（基于 D4 按 stdt_id 聚合）
**前端页面**：新建 `frontend/app/[locale]/referral-contributor/page.tsx`
