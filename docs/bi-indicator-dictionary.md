## 3. 基础达成

| 指标 | 定义 |
|------|------|
| Target | CC 当月基础标 |
| 金额 | MTD CC 签约金额 + 切割业绩金额 |
| 去小单付费金额 | MTD CC 大单（签约金额>=1500；CSCC>=1400）签约金额 |
| 切割业绩 | 有切割业绩时 CC 签约金额 |
| 激励虚拟业绩 | MTD CC 激励虚拟业绩汇总 |
| 判单虚拟业绩 | MTD CC 判单虚业绩汇总 |
| 退款金额 | MTD CC 退款金额汇总 |
| 净业绩 | `=金额 + 激励虚拟业绩 + 判单虚拟业绩 - 退款金额` |
| 达标人数 | `=MTD 净业绩达标率 >= BM`（同业绩是否达标） |
| 业绩是否达标 | `=if(完成率 >= BM, 1, 0)` |
| 完成率 | `=净业绩 / Target` |
| 100% | `=Target - 净业绩` |
| 110% | `=1.1 * Target - 净业绩` |
| 基础标净完成率_by业绩档位 | `=净业绩/Target` |
| 基础标净完成率_byCC | `=净业绩/Target` |
| 基础标净完成率_by工龄 | `=净业绩/Target` |
| 基础标净完成率_DGM KPI | `=净业绩/Target` |
| 达标人数占比 | `=SUM(业绩是否达标)/SUM(CC个人)` |
| 业绩达标率 | `=SUM(业绩是否达标)/SUM(CC个人)` |
| 人均付费金额 | `=AVERAGE(金额)` |
| 人均业绩（毛） | `=(金额+判单虚拟业绩)/CC个人` |
| 平均客单价(>1500) | `=去小单付费金额/去小单付费数` |

| B2C合同完成率(>1500) | `=去小单付费数/核定单量` |

## 11. MKT（市场渠道）

| 指标 | 定义 |
|------|------|
| 非推预约数 | `=leads数 - 推荐leads数` |
| 人均非推预约数 | `=AVERAGE(非推预约数)` |
| 非推出席/预约 | `=(出席数-推荐出席数)/(leads数-推荐leads数)` |
| 非推付费/预约 | `=(去小单付费数-推荐去小单付费数)/(leads数-推荐leads数)` |
| 非推荐付费金额MTD | `=金额-推荐付费金额` |

| MKTLeads占比 | `=(leads数-推荐leads数)/leads数` |
| MKT出席付费率 | `=(去小单付费数-推荐去小单付费数)/(出席数-推荐出席数)` |
| MKT平均客单价(>1500) | `=(去小单付费金额-推荐去小单付费金额)/(去小单付费数-推荐去小单付费数)` |
| MKT预约出席率 | `=(出席数-推荐出席数)/(leads数-推荐leads数)` |
| MKT预约付费率 | `=(去小单付费数-推荐去小单付费数)/(leads数-推荐leads数)` |
| MKT转化(去小单) | `=(去小单付费数-推荐去小单付费数)/(leads数-推荐leads数)` |

## 与 ref-ops-engine 字段映射说明

本字典中的指标与 ref-ops-engine 现有数据模型的对应关系：

| BI 指标类别 | ref-ops-engine 对应模块 | 覆盖状态 |
|------------|----------------------|---------|
| 转介绍（§1） | `src/analysis_engine.py` + `config.json indicator_registry` | 部分覆盖（渠道分类 SQL 可对齐 `channel_metric_scope`） |
| 基础达成（§3） | 待对接 BI 数据源 | 未覆盖 |
| REF（§10） | `backend/routers/data.py` REF 相关 API | 部分覆盖 |

> **注意**：`chnl4_name` 渠道分类 SQL 中的 `'lp_recommend.'`（带句号）疑为数据源脏数据，需与 BI 确认。
