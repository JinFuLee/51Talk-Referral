# 三 CLI 评分方法学附录

## raw_score(0..5) 判定规则

| 分值 | 规则 |
|---|---|
| 5 | 官方文档把该能力作为一等功能公开暴露，且直接命中当前阶段主目标 |
| 4 | 官方文档清楚支持该能力，但不是该阶段最差异化强项 |
| 3 | 能稳定完成，但需要更多手工拼装或不是默认路径 |
| 2 | 只具备部分能力，需明显额外上下文或外部工具 |
| 1 | 间接可达成，但不建议用于该阶段 |
| 0 | 在本次引证范围内没有官方支持证据 |

## 阶段权重选择依据

| 阶段 | 权重 | 依据 |
|---|---:|---|
| Research | 0.25 | 新鲜信息重要，但不直接等于交付 |
| Review | 0.30 | 决策正确性直接影响返工成本 |
| Delivery | 0.45 | SEE 以落地、验证、闭环为最高优先级 |

## criterion 权重原则

- `research`：优先实时搜索，其次 headless / checkpoint / permissions / MCP / 多入口面
- `review`：优先结构化评审输出，其次 MCP / 子代理 / SEE 规则适配 / 多入口面
- `delivery`：优先代码读改运行，其次权限边界 / 多入口面 / MCP / headless / checkpoint / SEE 规则适配

## 复用规则

- 评分先看官方证据，再看内部规则约束。
- 任何分数变动都必须同步更新 `scores_by_cli.csv` 与 `calc.md`。
