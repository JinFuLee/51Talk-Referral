## บทที่ 1: โครงสร้างองค์กรและคำจำกัดความตำแหน่ง {#ch1-org}

### 1.1 ลำดับชั้นองค์กร

| ระดับ | คำอธิบาย | ตัวอย่าง |
|-------|---------|---------|
| ประเทศ | ระดับสูงสุด | ประเทศไทย |
| แผนก | 3 แผนกหลัก CC / SS / LP | หน้าบ้าน CC, หลังบ้าน SS, บริการ LP |
| ภูมิภาค | เขตข้ามทีม | ภูมิภาคไทย |
| DGM | ผู้บริหารเขต | — |
| SD | ผู้อำนวยการฝ่ายขาย | — |
| SM | ผู้จัดการฝ่ายขาย | — |
| Team | ทีมย่อย | TH-CC01Team, TH-SS01Team |
| บุคคล | บัญชีส่วนตัว CC / SS / LP | THCC-patimakorn, THSS-thitima |

**การกรองข้อมูลตามภูมิภาค**: DataManager กรองทีมที่ไม่มีคำนำหน้า `TH-` ออกหลังโหลดข้อมูล มีผลกับ API ทั้งระบบโดยอัตโนมัติ — ห้ามกรองแยกในแต่ละ API

### 1.2 คำจำกัดความ 3 ตำแหน่งหลัก

| รหัส | ชื่อเต็ม | หน้าที่ | ชื่อในข้อมูล |
|------|---------|--------|------------|
| **CC** | CC หน้าบ้าน | ฝ่ายขายหน้าบ้าน รับผิดชอบหาลูกค้า ติดตาม และปิดการขาย KPI หลักคือยอดขายใหม่ + จำนวนออร์เดอร์ | CC (ไม่มีชื่อแทน) |
| **SS** | SS หลังบ้าน | ฝ่ายขายหลังบ้าน รับผิดชอบกระตุ้นนักเรียนเก่าให้แนะนำ SS แบบแคบ = ขอรหัสแนะนำจากผู้ใช้โดยตรง (SOP ง่าย) | **EA** (พบบ่อยในข้อมูล แมปอัตโนมัติเป็น SS) |
| **LP** | LP บริการ | ฝ่ายบริการหลังบ้าน คล้ายที่ปรึกษาการเรียน/อาจารย์ LP มีแคมเปญแบบกว้าง "同言同語" | **CM** (พบบ่อยในข้อมูล แมปอัตโนมัติเป็น LP) |

> **กฎเหล็ก**: แดชบอร์ด/รายงานใช้ **CC / SS / LP** เสมอ — EA แมปเป็น SS, CM แมปเป็น LP, ทีมที่มีชื่อ "-" ในข้อมูลแมปเป็น THCC

### 1.3 ฟิลด์สถานะบุคคล (จากพจนานุกรมตัวชี้วัด BI §2)

| ฟิลด์ | คำนิยาม |
|-------|---------|
| 代数 | จำแนกตามระดับตำแหน่ง CC ปัจจุบัน (บันได้การเติบโต) |
| 岗位 | ตำแหน่ง CC ในเดือนนั้น |
| 职级 | ระดับ CC ในเดือนนั้น |
| 是否休眠 | CC อยู่ในสถานะพักงานเดือนนั้นหรือไม่ |
| 是否计算 | CC เข้าร่วมการคำนวณผลงานเดือนนั้นหรือไม่ |
| 核定工龄 | อายุงานที่ได้รับรองในเดือนนั้น |
| 在职状态 | สถานะทำงาน/ลาออก CC เดือนนั้น |
| 及格线 | จัดประเภทตามช่วง BM เปรียบเทียบกับอัตราผลงานสะสม MTD CC |
| Benchmark (BM) | เกณฑ์มาตรฐานประจำเดือน (ใช้ตัดสินผ่านเกณฑ์หรือไม่) |
| CC个人 | สมาชิก CC ที่มีฟิลด์ 是否计算 = ใช่ |

### 1.4 คำจำกัดความบทบาท UserA / UserB

| บทบาท | คำอธิบาย |
|-------|---------|
| **UserA / นักเรียน A** | นักเรียนปัจจุบัน (ผู้แนะนำ) — ฝั่งที่ส่งลิงก์แนะนำออกไป |
| **UserB / นักเรียน B** | นักเรียนใหม่ที่ถูกแนะนำ — ฝั่งที่สมัครผ่านลิงก์ |

## บทที่ 2: โมเดลธุรกิจการแนะนำสมาชิก {#ch2-referral}

> แหล่งอ้างอิง: `docs/glossary.md` + `docs/bi-indicator-dictionary.md §1` + `projects/referral/config.json` (`enclosure_role_assignment` / `enclosure_role_narrow` / `enclosure_role_wide` / `channel_metric_scope`)

### 2.1 คำจำกัดความการแนะนำสมาชิก

การสมัครหนึ่งรายการถือว่าเป็น "การแนะนำสมาชิก" เมื่อตรงตาม**ข้อใดข้อหนึ่ง**ดังนี้:
1. ช่องทางระดับแรก = "转介绍" (การแนะนำสมาชิก)
2. ช่องทางระดับแรก ≠ ช่องทางแนะนำ แต่มี ID ผู้แนะนำ และ ID ผู้แนะนำ ≠ 0

### 2.2 การจำแนกช่องทางและการแมป SQL

```sql
CASE
  WHEN chnl4_name IN ('cc_recommend', 'dapan_chai_zhanghao')
    THEN 'CC窄口径'
  WHEN chnl4_name IN ('is_recommend')
    THEN 'SS窄口径'
  WHEN chnl4_name IN ('lp_recommend', 'lp_recommend.', 'dapan_ss_crm_haibao')
    THEN 'LP窄口径'
  ELSE '宽口径'
END
```

> **หมายเหตุ**: `lp_recommend.` (มีจุด) คาดว่าเป็นข้อมูลสกปรกจากแหล่งข้อมูล ได้ยืนยันกับฝ่าย BI แล้วให้คงการแมปเดิมไปยัง LP แบบแคบ

| ช่องทาง | คำอธิบาย | คุณภาพ | อัตราการชำระเงินอ้างอิง |
|--------|---------|--------|----------------------|
| CC แบบแคบ | ลิงก์พนักงาน CC ผูกบัญชี (แนะนำเชิงรุก) | สูง | ~32% |
| SS แบบแคบ | ลิงก์พนักงาน SS(EA) ผูกบัญชี (SOP ขอรหัสแนะนำ) | กลาง | ~28% |
| LP แบบแคบ | ลิงก์พนักงาน LP(CM) ผูกบัญชี (มีแคมเปญกว้าง "同言同語") | กลาง | — |
| แบบกว้าง | นักเรียนเก่า (UserA) แชร์ลิงก์เอง ไม่แบ่งแผนก | ต่ำ | — |

### 2.3 คำจำกัดความช่วงระยะเวลา (ล้อม)

ช่วงระยะเวลา = จำนวนวันนับจาก**วันที่ชำระเงินครั้งแรก** (วันชำระเงินคือ Day 0)

#### ช่วงระยะเวลาแบบนับวัน (แม่นยำ ใช้ในการวิเคราะห์ประจำวัน)

| รหัสช่วง | ช่วงวัน | เดือนที่สอดคล้อง | อัตราการมีส่วนร่วมทั่วไป | อัตราการเช็คอินทั่วไป |
|---------|--------|----------------|------------------------|---------------------|
| d0_30 | 0-30 วันหลังชำระเงิน | M0 (เดือนที่ชำระ) | **22.4%** | **96.5%** |
| d31_60 | 31-60 วัน | M1 | 10.5% | 44.0% |
| d61_90 | 61-90 วัน | M2 | 6.9% | 35.0% |
| d91_120 | 91-120 วัน | M3 | 3.8% (ค่าเฉลี่ย M3-M5) | 26.4% (ค่าเฉลี่ย M3-M5) |
| d121_180 | 121-180 วัน | M4-M5 | 3.8% | 26.4% |
| d181_plus | 181+ วัน | M6+ | **2.4%** | **19.2%** |

**ข้อสังเกตสำคัญ**: M0 คือช่วงทองของการแนะนำสมาชิก อัตราการมีส่วนร่วมลดลงอย่างรวดเร็วตามเวลา (M0 = 22.4% → M6+ = 2.4%) ค่าสัมประสิทธิ์การพาใหม่ M0 = 1.35, M1 สูงสุดที่ 1.47

### 2.4 เมทริกซ์ความรับผิดชอบช่วงระยะเวลา × ตำแหน่ง

#### แบบแคบ (พนักงานติดต่อนักเรียนเชิงรุก)

> ตารางด้านล่างเป็น**ค่าเริ่มต้น** การกำหนดจริงตาม Settings → คอก

| ช่วงระยะเวลา (วัน) | ช่วงระยะเวลา (เดือน) | ตำแหน่งรับผิดชอบ | ขอบเขตตัวชี้วัด |
|------------------|-------------------|-----------------|----------------|
| 0-90 วัน | M0-M2 | **CC** | full_funnel (สมัคร/นัด/เข้าร่วม/ชำระ/ยอดเงิน + อัตราแปลงทั้งหมด) |
| 91-120 วัน | M3 | **SS** | leads + ตัวชี้วัดกระบวนการ (อัตราการติดต่อ/เช็คอิน) + อัตราแปลง leads→CC ข้ามตำแหน่ง |
| 121+ วัน | M4+ | **LP** | leads + ตัวชี้วัดกระบวนการ (อัตราการติดต่อ/เช็คอิน) + อัตราแปลง leads→CC ข้ามตำแหน่ง |

#### แบบกว้าง (นักเรียนแชร์เอง)

> ตารางด้านล่างเป็น**ค่าเริ่มต้น** การกำหนดจริงตาม Settings

| ช่วงระยะเวลา (วัน) | ช่วงระยะเวลา (เดือน) | ตำแหน่งรับผิดชอบ |
|------------------|-------------------|-----------------|
| 0-90 วัน | M0-M2 | **CC** |
| 91-120 วัน | M3 | **SS** |
| 121-180 วัน | M4-M5 | **LP** |
| 181+ วัน | M6+ | **ฝ่ายปฏิบัติการ** |

> **แหล่งอ้างอิงการตั้งค่า**: `projects/referral/config.json` → `enclosure_role_narrow` (แบบแคบ) / `enclosure_role_wide` (แบบกว้าง) เขียนผ่านหน้า Settings, ฝั่ง frontend ใช้ `useWideConfig()` อ่านจาก API `/api/config/enclosure-role` — ห้าม hardcode

### 2.5 กฎการ귀속ตัวชี้วัดตามช่องทาง

| ช่องทาง | บทบาท | ขอบเขตตัวชี้วัดที่ใช้ได้ |
|--------|-------|----------------------|
| CC (full_funnel) | CC | สมัคร→นัด→เข้าร่วม→ชำระ→ยอดเงิน + อัตราแปลงทั้งหมด (กรวยเต็ม) |
| SS/LP (leads_and_process) | SS, LP | จำนวน leads + ตัวชี้วัดกระบวนการ (อัตราติดต่อ/เช็คอิน) + อัตราแปลง leads→CC (ประสิทธิภาพข้ามตำแหน่ง ไม่ใช่ KPI ตัวเอง) |
| แบบกว้าง (leads_only) | แบบกว้าง | จำนวน leads เท่านั้น |

**หมายเหตุพิเศษ**: "อัตราแปลง" ของ SS/LP = สัดส่วน leads ที่ SS/LP นำมาแล้วถูก CC แปลงเป็นการชำระเงิน (ประสิทธิภาพข้ามตำแหน่ง) — ไม่ใช่กรวยการขายของ SS/LP เอง

### 2.6 คำจำกัดความนักเรียนที่มีสิทธิ์

นักเรียนที่มีสิทธิ์ (กลุ่มพื้นฐานที่สามารถเข้าร่วมการแนะนำได้) ต้องผ่าน**ทุกข้อ**:
1. **ชำระเงินรายการใหญ่ครั้งแรก**: เคยชำระเงินครั้งแรกในรายการใหญ่แล้ว
2. **คูปองคอร์ส > 0**: มียอดคูปองคอร์สคงเหลือ
3. **แพ็กเกจยังไม่หมดอายุ**
4. **สถานะบัญชี = เปิดใช้งาน (on)**

### 2.7 การจำแนกประเภทนักเรียน

| รหัสประเภท | ชื่อเต็ม | คำอธิบาย |
|-----------|--------|---------|
| aa | อเมริกันคลาสเล็ก | นักเรียนคอร์สกลุ่มเล็กภาษาอังกฤษสไตล์อเมริกัน |
| k12 | เยาวชน | นักเรียนในระดับ K12 |
| non-k12 | ผู้ใหญ่ | นักเรียนผู้ใหญ่ |

มาตรฐานการจำแนกของไทยเหมือนกับระดับโลก

### 2.8 รายละเอียดการเข้าร่วมแนะนำของ SS/LP

- **SS แบบแคบ**: SS ขอรหัสแนะนำจากผู้ใช้โดยตรง (SOP ง่าย ขอตรงๆ)
- **LP แบบกว้าง**: แคมเปญ "同言同語" — ผู้ใช้อ่านตามในแอปแล้วแชร์ในกลุ่มชุมชนเพื่อเช็คอิน ข้อมูลเก็บในระบบ BI อีกตัว รอ API ก่อนนำมาผสาน

### 2.9 สูตรการแนะนำสมาชิก

```
ผลผลิตการแนะนำ = ผู้ใช้ที่ active × อัตราการมีส่วนร่วม × อัตราการรับสมาชิกใหม่ × อัตราแปลง
```

**ระยะวิวัฒนาการ**: เริ่มต้นพื้นฐาน (ขับเคลื่อนด้วยแรงจูงใจ) → การดำเนินงานเชิงวิทยาศาสตร์ (ตามสูตร) → การคิดเชิงระบบ (บริหารสต็อก 2 ชั้น)

## บทที่ 3: พจนานุกรม KPI {#ch3-kpi}

> แหล่งอ้างอิง: `docs/bi-indicator-dictionary.md` + `docs/glossary.md` + `docs/research/key-metrics-quick-reference.md`

### 3.1 ตัวชี้วัดระดับนักเรียน

**มาตรฐานการโทรที่มีประสิทธิผลกำหนดเป็น >=20s** (อัตราการติดต่อ/เวลาโทร/จำนวนครั้งโทรใช้เกณฑ์นี้ทั้งหมด ปรับแก้จาก 120s ในเอกสารเดิม)

| ชื่อตัวชี้วัด | สูตร | หน่วย | หมายเหตุ |
|------------|-----|------|--------|
| **อัตราการติดต่อ** | จำนวนนักเรียนที่รับสายสำเร็จในเดือน / จำนวนนักเรียนที่มีสิทธิ์ในเดือน | % | รับสายสำเร็จ = ระยะเวลาโทร **≥20s** |
| **อัตราการมีส่วนร่วม** | จำนวนนักเรียนที่นำ ≥1 การสมัครแนะนำในเดือน / จำนวนนักเรียนที่มีสิทธิ์ในเดือน | % | วัดความ active ของการแนะนำสมาชิก |
| **อัตราเช็คอิน** | จำนวนนักเรียนที่สร้างลิงก์และแชร์ในเดือน / จำนวนนักเรียนที่มีสิทธิ์ในเดือน | % | สร้างลิงก์ = สร้าง/เปิดใช้งาน QR/ลิงก์การแนะนำเฉพาะตัว |
| **ค่าสัมประสิทธิ์การพาใหม่** | จำนวนนักเรียน B ที่สมัครในเดือน / จำนวนนักเรียน A ที่นำการสมัครในเดือน | เท่า | แต่ละผู้แนะนำพามาเฉลี่ยกี่คน |
| **อัตราส่วนนำใหม่** | จำนวนผู้สมัครที่แนะนำในเดือน / จำนวนนักเรียนที่มีสิทธิ์ในเดือน | % | ประสิทธิภาพผลผลิตโดยรวมของกลุ่ม |
| **จำนวนการสมัครใหม่** | จำนวนการสมัคร B ที่นักเรียน A นำมาในเดือน | คน | ค่าสัมบูรณ์ |
| **อัตราการโทร** | จำนวนนักเรียนที่ CC โทรหาในเดือน / จำนวนนักเรียนที่มีสิทธิ์สิ้นเดือน | % | ความครอบคลุมการติดตาม |

### 3.2 ตัวชี้วัดกรวย

```
สมัคร → นัด → เข้าร่วม → ชำระเงิน
```

| ชื่อตัวชี้วัด | สูตร | หน่วย | หมายเหตุ |
|------------|-----|------|--------|
| **สมัคร (Leads)** | จำนวนนักเรียน B ที่สมัครผ่านลิงก์แนะนำ | คน | ทางเข้ากรวย |
| **จำนวนนัด** | จำนวนคนนัดคอร์สทดลอง | คน | นัดครั้งแรก ไม่นับซ้ำ |
| **จำนวนเข้าร่วม** | จำนวนคนเข้าคอร์สทดลอง | คน | เข้าครั้งแรก ไม่นับซ้ำ |
| **จำนวนชำระเงิน** | จำนวนคนที่ชำระเงินสำเร็จ | คน | ตัด small order: มูลค่าสัญญา ≥1500 USD (CSCC ≥1400) |
| **อัตราการนัด** | จำนวนนัด / จำนวนสมัคร | % | Appt% |
| **อัตราเข้าร่วมจากนัด** | จำนวนเข้าร่วม / จำนวนนัด | % | Show up% |
| **อัตราชำระจากเข้าร่วม** | จำนวนชำระ / จำนวนเข้าร่วม | % | Show up to pay% |
| **อัตราชำระจากสมัคร** | จำนวนชำระ / จำนวนสมัคร | % | อัตราแปลงจากต้นถึงปลาย |
| **ราคาต่อลูกค้า (ASP)** | ยอดชำระ (USD) / จำนวนออร์เดอร์ | USD | เกณฑ์ big order ≥1500 USD |

**ข้อมูลตัวอย่าง 2026-01**: สมัครรวม 582, นัด 440, เข้าร่วม 266, ชำระ 113, ยอด 111,636 USD; อัตรานัด 75.6%, อัตราเข้าร่วมจากนัด 60.5%, อัตราชำระจากเข้าร่วม 42.5%, อัตราแปลงรวม 19.4%

### 3.3 ผลงานพื้นฐาน

**ผลงานของระบบนี้ = มูลค่าสัญญา (ตัด small order) ไม่รวมค่าเทียมกระตุ้น / ค่าเทียมตัดสิน / ยอดคืนเงิน**

| ชื่อตัวชี้วัด | สูตร/นิยาม | หน่วย |
|------------|----------|------|
| Target | เป้าพื้นฐานรายเดือน CC | USD |
| ยอดขาย (ยอดรวม) | มูลค่าสัญญา MTD CC + มูลค่าส่วนแบ่ง | USD |
| ยอดชำระ big order | มูลค่าสัญญา big order (≥1500; CSCC ≥1400) | USD |
| ค่าเทียมกระตุ้น | ยอดค่าเทียมกระตุ้นสะสม MTD CC | USD |
| ค่าเทียมตัดสิน | ยอดค่าเทียมตัดสินสะสม MTD CC | USD |
| ยอดคืนเงิน | ยอดคืนเงินสะสม MTD CC | USD |
| **ผลงานสุทธิ** | ยอดขาย + ค่าเทียมกระตุ้น + ค่าเทียมตัดสิน - ยอดคืนเงิน | USD |
| อัตราบรรลุเป้า | ผลงานสุทธิ / Target | % |
| ผ่านเกณฑ์หรือไม่ | if(อัตราบรรลุ ≥ BM, 1, 0) | Boolean |
| สัดส่วน/อัตราผ่านเกณฑ์ผลงาน | SUM(ผ่านเกณฑ์) / SUM(CC บุคคล) | % |
| ราคาต่อลูกค้าเฉลี่ย (>1500) | ยอดชำระ big order / จำนวน big order | USD |

### 4.3 รายละเอียดอัลกอริทึมสำคัญ

| กลไก | คำอธิบาย |
|-----|---------|
| **min-max normalization** | (v - min) / (max - min) ช่วง [0, 1]; ถ้าทุกคนค่าเท่ากัน normalize เป็น 0.5 |
| **_redistribute** | CC: เมื่อข้อมูลขาด น้ำหนักกระจายตามสัดส่วนไปยังมิติอื่นในประเภทเดียวกัน ทำให้คะแนนประเภทยังเต็มที่ 1.0 |
| **_redistribute_role** | SS/LP: เหมือนกัน แต่กระจายภายในบทบาทเดียวกัน |
| **paid_share** | ตัวชี้วัดการมีส่วนร่วม SS/LP = จำนวนชำระ CC ของบุคคล / จำนวนชำระ CC รวมของทีมบทบาทเดียวกัน |
| **leads_to_cc_rate** | ตัวชี้วัดคุณภาพ SS/LP = ฟิลด์ `conversion_rate` ของ A4 หรือ paid/leads คิดใน scope เดียวกับ CC |

### 4.4 ตารางเปรียบเทียบ CC กับ SS/LP

|-----|---------------------|----------------------|
| กระบวนการ | 7 มิติ (โทรออก/รับสาย/รับสายผล/ติดตาม 4 แบบ) | 2 มิติ (อัตราติดต่อ/เช็คอิน) |
| ผลลัพธ์ | 7 มิติ (สมัคร/leads/ผู้ใช้แนะนำ/ราคา/ออร์เดอร์/ผลงาน/สัดส่วน) | 1 มิติ (จำนวน leads) |
| ประสิทธิภาพ | 4 มิติ (อัตราแปลง/เช็คอิน/มีส่วนร่วม/พาใหม่) | ไม่มีประเภทประสิทธิภาพแยก |
| คุณภาพ | ไม่มีประเภทคุณภาพแยก | 1 มิติ (อัตราแปลง leads→CC) |
| การมีส่วนร่วม | สัดส่วนผลงานอยู่ในประเภทผลลัพธ์ | ประเภทการมีส่วนร่วมแยก (paid_share) |
| น้ำหนักรวม | process×0.25 + result×0.60 + efficiency×0.15 | process×0.25 + result×0.30 + quality×0.25 + contribution×0.20 |
| ช่วง normalize | normalize ในกลุ่ม CC ทั้งหมด | SS และ LP normalize อิสระแยกกัน |

### 5.2 ระบบส่วนต่างคู่

KPI เชิงตัวเลขทุกตัวต้องคำนวณ**ส่วนต่าง 2 แบบ + ค่าเฉลี่ยต่อวัน 2 แบบ**:

| ประเภทส่วนต่าง | สูตร | ความหมาย |
|--------------|-----|---------|
| **ส่วนต่างเป้าสัมบูรณ์** | `actual - target` | ห่างจากเป้ารายเดือนเท่าไร (ติดลบ=ล้าหลัง, บวก=เกิน) |
| **ส่วนต่างความคืบหน้าเวลา** | `actual/target - time_progress` | ตามทันความคืบหน้าตามเวลาหรือไม่ (ติดลบ=ตกขบวน) |
| **ค่าเฉลี่ยต่อวันที่ต้องการบรรลุเป้า** | `(target - actual) / remaining_workdays` | ต้องทำต่อวันเท่าไรจึงจะถึงเป้ารายเดือน |
| **ค่าเฉลี่ยต่อวันที่ต้องการตามทันความคืบหน้า** | `max(0, target × time_progress - actual) / remaining_workdays` | ต้องทำต่อวันเท่าไรจึงจะตามทันเส้นความคืบหน้า |

**ฟิลด์ backend ที่สอดคล้อง**: `absolute_gap` (ส่วนต่างเป้า), `gap` (ส่วนต่างความคืบหน้า), `remaining_daily_avg` (ค่าเฉลี่ยบรรลุเป้า), `pace_daily_needed` (ค่าเฉลี่ยตามทันความคืบหน้า)

### 7.3 กลยุทธ์ fallback ข้อมูลต้นทุน ROI 3 ระดับ

เมื่อข้อมูลต้นทุน ROI ไม่พร้อมใช้ ใช้ลำดับดังนี้:

```
① ต้นทุนจริง (ไฟล์ต้นทุน ROI)
    ↓ ไม่พร้อมใช้
② ประมาณการจากการกระจายรายละเอียดออร์เดอร์ (คำนวณค่าคอมมิชชันจากสัดส่วน small/big order)
    ↓ ไม่พร้อมใช้
③ สมมติ 50/50 (small order : big order แต่ละครึ่ง)
```

เมื่อ fallback ให้แสดงความน่าเชื่อถือ 🟡 (โหมดประมาณการ) ในรายงาน

## บทที่ 10: วิธีวิทยาการวิเคราะห์ {#ch10-methodology}

### 10.1 กระบวนการวิเคราะห์ข้อมูล 6 ขั้นตอน

โมดูลการวิเคราะห์ทั้งหมดปฏิบัติตาม:

| ขั้นตอน | ชื่อ | การดำเนินการหลัก |
|--------|-----|---------------|
| 1 | **ชี้แจงคำถาม** | คำถามหลัก + ผู้เกี่ยวข้อง + คุณค่าหลักที่การวิเคราะห์นี้จะสร้าง |
| 2 | **ตัวชี้วัดสำคัญ** | ตัวชี้วัดผลลัพธ์ (สมัคร/ชำระ/รายได้) + ตัวชี้วัดกระบวนการ (เช็คอิน/ติดต่อ/มีส่วนร่วม) |
| 3 | **แหล่งข้อมูลสนับสนุน** | แหล่งข้อมูล (12 แหล่ง) + เครื่องมือประมวลผล (AnalysisEngineV2) |
| 4 | **วิธีวิเคราะห์** | วิธีหลัก (กรวย/การเปรียบเทียบ/การ귀속) + วิธีเสริม (ตรวจจับความผิดปกติ/พยากรณ์) |
| 5 | **ข้อค้นพบสำคัญ** | วิเคราะห์สาเหตุรากเหง้า + ผลการวิจัยสำคัญ |
| 6 | **แผนปฏิบัติ** | คำแนะนำเชิงกลยุทธ์ + ผลกระทบที่คาดหวัง (วัดเป็น $) |

### 10.2 หลักการพีระมิดและกรอบรายงาน

- **สรุปก่อน**: ให้ข้อสรุปหลักก่อน แล้วขยายหลักฐาน
- **หลัก MECE**: แยกอย่างอิสระ ครอบคลุมทั้งหมด
- **กรอบ SCQA**: บริบท (S) → ความขัดแย้ง (C) → คำถาม (Q) → คำตอบ (A)
- แต่ละชั้นข้อโต้แย้ง 3-7 ข้อ ไม่เกิน 7
- ตรรกะก้าวหน้า: ตามลำดับเวลา / ตามพื้นที่ / ตามระดับ

### 10.3 โมเดลวิวัฒนาการ 3 ระยะของการแนะนำสมาชิก

| ระยะ | แรงขับเคลื่อนหลัก | คุณลักษณะสำคัญและเครื่องมือ |
|-----|-----------------|--------------------------|
| **1. เริ่มต้นพื้นฐาน** | ความตั้งใจของผู้ใช้ (แรงจูงใจเป็นหลัก) | แรงจูงใจผลลัพธ์ (60 USD/20 คาบ), ความสามารถเครื่องมือ (VIP/บัตรมิตรภาพ/ผู้ช่วยน้อย/รหัสแนะนำ), ส่วนได้เสีย 4 ประการ (ตลาด/เริ่มต้น/สต็อก/การขาย) |
| **2. การดำเนินงานเชิงวิทยาศาสตร์** | การดำเนินงานตามสูตร | สูตรการแนะนำ = ผู้ใช้ active × อัตราการมีส่วนร่วม × อัตรารับใหม่ × อัตราแปลง; หลายช่องทางละเอียด (ขอโดยตรง/เช็คอิน/ไลฟ์/ชุมชน/พาร์ทเนอร์/ความร่วมมือกับโรงเรียน) |
| **3. การคิดเชิงระบบ** | บริหารสต็อก 2 ชั้น | สต็อกที่ 1: ผู้ใช้ active ที่พึงพอใจ = คุณภาพผลิตภัณฑ์ + ประสบการณ์บริการ = ความพึงพอใจผู้ใช้; สต็อกที่ 2: เครือข่ายสังคมผู้ใช้ (ความไว้วางใจสูง/ความต้องการสูง, ความไว้วางใจสูง/ความต้องการต่ำ, ความไว้วางใจต่ำ/ความต้องการสูง) |

### 10.4 โมเดลเหตุผลการดำเนินงานแนะนำสมาชิก

```
01 ตัวขับธุรกิจ (สาเหตุ)        02 การรับรู้ผู้ใช้ (รากฐาน)     03 การก้าวระดับ (ผล)
├─ ระบบ: ความแม่นยำ+ที่ทำงาน   ├─ ความตั้งใจ: กลยุทธ์+รางวัล   ผู้ใช้ชำระทั้งหมด
├─ บริการผู้ใช้: โปสเตอร์+ข้อความ+รางวัล  ├─ ความสามารถ: ทักษะ+แชร์ง่าย  ↓ ผู้ใช้ที่เข้าร่วมแนะนำ
├─ คน: CC ต้องการ/ทำได้หรือไม่  ├─ สิ่งแวดล้อม: หลากสถานการณ์      ↓ ผู้ใช้แนะนำที่ active
└─ กลยุทธ์: นโยบายสนับสนุน      └─ ผลิตภัณฑ์: ง่ายต่อการใช้          ↓ พาร์ทเนอร์
```

**ตัวคันโยกสำคัญ**: คน (ความตั้งใจและความสามารถของ CC) > ผลิตภัณฑ์ (ง่ายต่อการใช้) > กลยุทธ์ (แรงจูงใจนโยบาย)

### 10.5 การวิเคราะห์ 5-Why แบบพีระมิด

บริบทที่ใช้: เรียกอัตโนมัติเมื่อตัวชี้วัดเบี่ยงเบน >2σ

```
ผลลัพธ์ผิดปกติ (เช่น อัตราการมีส่วนร่วม -30%)
  → Why 1: อัตราเช็คอินลดลง? (ข้อมูลสนับสนุน: เช็คอิน 70% → 55%)
    → Why 2: รางวัลกิจกรรมเช็คอินลดลง? (ยืนยันการเปลี่ยนแปลงนโยบายแรงจูงใจ)
      → Why 3: งบประมาณถูกตัด? (ยืนยันนโยบายต้นทุน)
        → Why 4: ROI ไม่ถึงเป้าทำให้เกิด? (ยืนยันข้อมูล ROI)
          → Why 5: อัตราแปลงลดลงดึง ROI ต่ำ (สาเหตุรากเหง้า)
            → แผนปฏิบัติ: ปรับ SOP การติดต่อ M0 + เพิ่มอัตราชำระจากเข้าร่วม
            → คาดหวัง: อัตราการมีส่วนร่วม +1.5pp, พาใหม่ +45 คน/เดือน, ผลงาน +$38K/เดือน
```
